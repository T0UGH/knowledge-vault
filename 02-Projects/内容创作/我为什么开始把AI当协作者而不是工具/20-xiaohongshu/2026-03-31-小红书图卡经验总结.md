# 小红书图卡经验总结：我为什么开始把 AI 当协作者，而不是工具

## 结论

这次小红书图卡整体是对的。

不是因为文案写得多，而是因为几件关键事做对了：
- 先把观点收成 **6 张卡的结构**，而不是直接堆长文
- 先定 **hook + 节奏 + 每张卡任务**，再写输入稿
- 又单独做了一版 **短句版输入稿**，明显更适合图卡生成
- 产物最后收进了统一项目路径，而不是散落在单文件层

## 这次有效的做法

### 1. 先做结构，不先抠句子

这篇内容最开始是公众号文章思路。
真正转成小红书之后，最关键的一步不是“压缩字数”，而是先把它改写成：
1. 封面观点卡
2. 结论卡
3. 变化一
4. 变化二
5. 变化三
6. 收尾互动卡

这个结构一旦立住，后面不管是写文案还是生成图片，都不会乱。

### 2. hook 是第一优先级

这次封面句子：

> 把 AI 当工具，已经不够了

这是整组图能成立的核心。

原因很简单：
- 有立场
- 有冲突
- 一眼能懂
- 能把“为什么不够了”这个阅读动力拉出来

做小红书图卡时，封面不是摘要，封面是拦截器。

### 3. 图卡文案和文章文案不是一回事

长文版写得顺，不代表适合出图。
这次后面专门整理了一版《XHS短句版输入稿》，这一步很值。

图卡更适合：
- 短句
- 对照句
- 重复句式
- 每张卡只讲一个小判断

比如这组里最有效的一类句子就是：
- 工具帮我处理一条信息
- 协作者开始帮我持续处理一个信息流

这种句子天然适合放大、分层、做海报感。

### 4. 观点型图卡，视觉应该服务文字，不要反过来

这次判断是对的：
- 适合轻海报感
- 适合 screen-print / bold 一类有观点张力的风格
- 适合 balanced 布局
- 不适合萌系、种草风、超写实产品图

原因不是审美问题，而是信息类型不一样。
这篇本质上是“认知转变 + 方法表达”，不是教程，也不是生活方式展示。
如果画面太花、太可爱、太写实，都会削弱观点力度。

### 5. 目录结构比想象中重要

这次后面把内容从零散文件收进：

- `00-source/`
- `10-wechat/`
- `20-xiaohongshu/`
- `20-xiaohongshu/xhs-images/ai-collaborator/`

这个整理动作很有价值。

因为内容创作不是一次性交付，而是后续还会继续：
- 改稿
- 衍生微博 / 公众号 / 小红书版本
- 复盘哪种表达更有效
- 复用 prompt / 图卡结构 / 视觉判断

如果路径一开始不收干净，后面每次继续都会重新找文件。

## 这次暴露出来的坑

### 1. 生成链路依赖不够前置

之前一度以为图片还没真正生成成功，但现在看目录里 6 张 png 已经都在了。
这说明流程里“状态记录”和“实际产物状态”有点脱节。

后面更稳的做法：
- 每次生成后，明确记录“是否已生成到文件”
- `GENERATION-STATUS.md` 不只记失败，也要记最终成功状态
- 最好顺手记一下使用的模型 / 风格 / 是否满意

否则过两天回头看，很容易分不清“准备好了”和“已经跑完了”。

### 2. 从旧路径迁到新路径时，容易留下 git 噪音

这次仓库里出现了：
- 旧文件删除
- 新目录未提交
- `.DS_Store`
- `.baoyu-skills/`
- `.obsidian/graph.json`

这类东西本身不难处理，但会污染提交判断。

经验是：
- 提交时只 stage 这次真正的内容路径
- 不要顺手把 Obsidian 运行态和本地缓存带上
- 路径重构最好配一条明确 commit message，避免以后看历史看不懂

### 3. 平台输入稿需要单独维护，不要幻想一稿通吃

同一个主题可以共享核心观点，但：
- 公众号需要展开
- 小红书需要短句节奏
- 微博可能需要 punchline + 轻展开

真正高效的做法不是“一稿多发”，而是：
- 保留一个 source 层
- 针对平台各出一版投放稿

## 可复用的方法论

后面再做观点型小红书图卡，建议固定用这个顺序：

1. 先写 source / 提纲
2. 抽成 5~7 张卡的结构
3. 给每张卡定义“任务”
4. 再压一版短句输入稿
5. 最后再做图像生成
6. 产物、prompt、经验统一归档

一句话概括：

> 小红书图卡不是“把长文压短”，而是“把一个观点重新设计成滑动叙事”。

## 这次最值得保留的经验

如果内容本身是“认知变化”而不是“技巧教程”，那最有效的表达方式通常不是信息更密，而是：
- 观点更清楚
- 对照更鲜明
- 句子更短
- 节奏更稳

这类内容真正拼的不是知识量，而是读者刷到第一页时，愿不愿意继续滑下去。

## 下一步建议

1. 选 1~2 张最满意的图，倒推总结一下哪些画面元素最有效
2. 补一条发布版 caption / 配文
3. 如果这组数据反馈不错，把这个流程抽成固定模板，后面直接复用
