# 内容创作

## Included
- 以“主题”为单位组织的长期内容工作区
- 每个主题下的 source / 微信稿 / 小红书稿 / 后续多渠道改写稿
- 与单个主题强绑定的结构草图、具体场景、封面素材、渠道版本

## Structure Rule
内容创作体系已拍板为：

- 主真身统一放在 `02-Projects/内容创作/`
- 一个主题一个目录
- 每个主题目录内部固定按以下结构组织：

```text
<主题目录>/
  README.md
  00-source/
  10-wechat/
  20-xiaohongshu/
```

详细规则见：
- [[00-Home/2026-04-06-content-creation-structure-decision]]
- [[00-Home/2026-04-06-obsidian-vault-cleanup-review]]

## Usage Rule
- **一个主题只有一个长期真身目录**
- 不再让渠道目录（例如旧 `公众号创作`）长期持有第二套正文真身
- 如果未来需要“作品总览”，优先做索引页，而不是重新建立一层正文目录
