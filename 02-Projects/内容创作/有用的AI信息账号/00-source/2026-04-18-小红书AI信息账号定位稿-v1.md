# 小红书AI信息账号定位稿 v1

## 一句话定位

> 帮人看懂 AI 工具、Agent、知识库、工作流背后的真实变化，并给出普通人现在就能用上的判断与做法。

## 这不是一个什么号

这个号不应该做成：

1. **热点资讯搬运号**
   - 追新闻、追融资、追模型榜单
   - 更新快，但内容资产弱
   - 很容易卷成媒体快讯

2. **纯教程说明书号**
   - 全是“手把手教你”“三分钟学会”
   - 收藏可能高，但长期同质化严重
   - 容易把账号做成工具文档

3. **强人格陪伴号**
   - 重点放在个人状态、个人生活、日常表达
   - 会把账号拉向“我这个人”，而不是“这个号持续能提供什么价值”

## 这应该是一个什么号

更适合做成：

> **AI 工作方式情报与转译账号**

核心不是“更快报新闻”，也不是“更像朋友”，而是：

- 更会筛选
- 更会解释
- 更会把内部材料转成平台可读内容
- 更会给出稳定判断，而不是只给情绪和热闹

## 核心承诺

这个号应该向用户提供的，不是“我最近在想什么”，而是：

1. **这次变化到底发生了什么**
2. **它真正重要在哪一层**
3. **普通人现在应该怎么理解 / 怎么用 / 要不要跟**

也就是说，账号的价值应当是：

> 帮用户省时间、少踩坑、快速跟上真正值得关注的 AI 工作方式变化。

## 目标受众

最适合的不是泛路人，而是以下三类人：

### 1. 已经在用 AI，但还没有形成稳定工作流的人
他们会用 ChatGPT、Claude、Cursor 一类工具，但经常处于：

- 用得到一点
- 但接不进长期工作流
- 工具越装越多，结构反而更乱

### 2. 对 Agent / Memory / Knowledge Base / 自动化感兴趣的人
他们已经开始意识到：

- 光会聊天不够
- AI 真正的价值在于进入工作结构
- 需要更稳定的外部脑与执行链路

### 3. 不想追所有热点，只想知道“什么真的值得学”的人
他们不是不关心新变化，而是：

- 不想被概念轰炸
- 不想天天切新工具
- 想知道什么是长期有复利的东西

## 差异化来源

这个号最该占的坑，不是“AI 新闻”，也不是“vibe coding 审美作品”，而是：

> **知识库 / Memory / Skill / Agent Workflow 的中文稳定转译坑。**

原因：

1. 这类内容已经有需求，但还没有被完全占满。
2. 它天然适合图卡、结构拆解、最佳实践和关系图表达。
3. 它比单纯换模型、换工具更有长期复用价值。
4. 它是真正“有用”的内容，用户可以据此调整自己的工作方式。

## 内容主轴

这个号的母题应该统一成一句话：

> **AI 正在如何改写人的工作接口、知识组织方式和执行方式。**

以后所有内容，最好都能挂回这个母题，而不是东一篇西一篇。

## 栏目设计

建议前期先固定为 3 个主栏目。

### 栏目 1：这次变化，真正重要在哪

这是一组“判断型内容”。

适合回答：

- Claude Code 这次更新，重要的不是功能，而是接口变了
- MCP 现在到了哪个阶段：为什么科普期已经过去了
- 为什么 AI 知识库会成为下一阶段刚需
- 很多人其实理解错了 Agent，它不是自动化脚本升级版

这一栏的作用是让用户形成认知：

> 这个号不是在报新闻，而是在帮我判断。

### 栏目 2：真实可用的工作流 / 最佳实践

这是一组“复用型内容”。

适合回答：

- 一个能长期用的 AI 工作流，至少要有哪几层
- Claude Code / Codex / OpenClaw 分别更适合放在哪一层
- 我怎么让 AI 真正记住长期信息，而不是每次重新聊
- 什么样的 Skill 值得长期维护，什么样只是一次性 prompt

这类内容不要写成说明书，而应该写成：

- 场景
- 结构
- 原则
- 推荐做法
- 常见误区

### 栏目 3：把复杂内部材料翻译成人能看懂的话

这是一组“转译型内容”。

适合回答：

- Memory、Knowledge Base、Skill 到底是什么关系
- 为什么很多人其实理解错了 MCP
- “会用 AI”和“能把 AI 接进工作流”差的到底是什么
- 现在最值得搭的不是第二大脑，而是可持续外部脑

这一栏是长期差异化来源，因为它要求真正理解内部结构，不能只靠改写新闻稿。

## 内容比例建议

前 15 到 20 篇，不建议太散。建议按下面的比例来：

- **40% 判断型内容**：讲“为什么重要”
- **40% 工作流型内容**：讲“怎么长期用”
- **20% 世界观内容**：讲“为什么我会这样判断”

其中世界观内容可以保留，但只适合做辅助层，不适合做主轴。

## 活人感怎么处理

这个号可以牺牲一部分活人感，但不能完全没有“人”。

### 建议保留的活人感

- 我现在怎么判断
- 我踩过什么坑
- 我为什么改成这套结构
- 这套方法是我自己长期在跑的

### 不建议强化的活人感

- 过多日常状态输出
- 过多生活化聊天
- 反复强调“我这个人”的情绪和经历
- 把自己做成账号最主要的消费对象

目标不是把账号做得像朋友，而是做得像：

> **一个可信的长期信息源。**

## 视觉与形式建议

最适合的主形式不是自拍视频，也不是强口播，而是：

- 图卡
- 结构化封面
- 6 到 9 页讲清一个问题
- 每页一句主判断

原因不是平台偏好，而是这个账号真正的优势在于：

- 结构
- 判断
- 转译
- 关系拆解

这些能力更适合图卡表达。

## 标题策略

建议优先使用“对象 + 判断 + 收益”的结构。

### 推荐句型

- 不是 X，而是 Y
- 一篇讲明白
- 很多人其实理解反了
- 它真正改变的不是功能，而是接口
- 值不值得折腾？我给一个真实判断
- 从“会用”到“能接进工作流”，差的到底是什么

### 例子

- Claude Code 这次更新，真正变的不是功能，而是接口
- 很多人其实理解错了 MCP：它的价值不只是“连工具”
- Memory、Knowledge Base、Skill，到底是什么关系？一篇讲明白
- 如果你只想跟一类 AI 变化，我建议你盯“工作方式”而不是“模型榜单”

## 首页呈现原则

一个新用户点进账号首页之后，应该迅速看明白这个号常发的是：

- 这件事为什么重要
- 这工具值不值得学
- 这套工作流怎么搭更稳
- 这概念很多人到底哪里理解错了
- 这类知识库 / Memory / Skill / Agent Workflow 怎么区分

如果用户进来后，只感觉“这个人很懂”，但说不清“这个号长期会提供什么”，那定位就还没有立住。

## 起号原则

前期最重要的不是追一条大爆，而是让平台和用户都知道：

> **这个号到底稳定输出什么。**

因此前期要避免：

- 题材过散
- 形式频繁切换
- 一会儿新闻，一会儿生活，一会儿教程，一会儿观点

更适合用固定栏目和统一标题结构，把账号味道先做出来。

## 当前最推荐的最终定位

如果现在要拍板，我的建议是：

> **做一个以 AI 工作流 / Agent / 外部大脑 / 知识组织为核心，持续提供判断、结构和最佳实践的有用账号。**

它不是人格博主号，不是热点资讯号，也不是纯教程号。

它应该成为：

> **一个帮助用户理解 AI 工作方式变化，并据此调整自己工作结构的账号。**

## 下一步

在这个定位基础上，后续最值得继续补的不是抽象讨论，而是把它继续做实成：

1. 账号名 / 简介候选
2. 前 12 篇起号选题
3. 首页栏目与封面风格规范
4. 图卡模板与标题模板
5. “什么能发 / 什么不发”的选题边界清单
