# 小红书AI信息账号前12篇起号选题 v2

## 这版为什么重做

v1 的问题不是标题不够好，而是内容引擎选错了。

v1 更像：

- 知识讲解号
- 方法论号
- 工作流 evergreen 内容号

但这个账号真正要做的不是“我来系统教你一套 AI 方法论”，而是：

> **我持续帮你筛选 AI 世界里真正重要的变化，然后讲清楚它为什么重要。**

所以 v2 要从“知识驱动”改成“变化驱动”。

## v2 的内容原则

新版 12 篇都尽量满足这 5 条：

1. **由最近变化 / 最近信号触发**，不是从抽象体系出发
2. **每篇都回答“所以呢，为什么重要”**
3. **以转译为主，不以课程化讲解为主**
4. **只补必要背景，不长成完整教材**
5. **首页看起来像高密度信息源，而不是 AI 方法论博主**

## v2 的首页结构

建议首页前 12 篇按这 4 类来排：

- 4 篇「最近最值得看的变化」
- 4 篇「这件事为什么重要」
- 2 篇「变化背后说明了什么」
- 2 篇「补一张背景卡」

这样首页观感会更像：

> 一个持续筛选变化、给判断、补背景的信息号。

而不是：

> 一套按章节展开的 AI 教程。

## 首页前 12 篇清单 v2

## 01｜这件事为什么重要

### 标题
**AI 正在从“回答问题”变成“接入工作”**

### 核心判断
这篇不是抽象讲历史，而是给用户一个当前视角：

- 最近热的，不再只是模型能力
- 真正持续出现的新东西，是 Agent、Memory、Skill、Knowledge Base、工作流接口
- 这说明 AI 内容正在从“更会回答”转向“更能进入工作”

### 为什么它适合做首页第 1 篇
因为它既不是热点搬运，也不是纯知识卡，而是整个账号的母判断。

### 图卡页数建议
- **7 页**

### 图卡结构建议
1. 封面：AI 正在从回答问题变成接入工作
2. 最近热的内容类型变了什么
3. 为什么模型榜单不再是唯一主角
4. Agent / Memory / Skill / KB 为什么一起冒头
5. 这说明变化发生在哪一层
6. 对普通人意味着什么
7. 收口：这个号接下来就盯这种变化

---

## 02｜最近最值得看的变化

### 标题
**这周最值得看的 3 个 AI 工作流变化**

### 核心判断
这是情报号该有的第一篇“周度精选卡”。

重点不是堆链接，而是用 3 个变化告诉用户：

- 最近这条线在往哪里走
- 哪些变化值得理解
- 哪些只是热闹，不必全部追

### 为什么它适合做首页第 2 篇
因为它直接告诉用户：

> 这个号会持续帮我筛变化，而不是发散乱感想。

### 图卡页数建议
- **8 页**

### 图卡结构建议
1. 封面
2. 本周变化 1
3. 为什么重要
4. 本周变化 2
5. 为什么重要
6. 本周变化 3
7. 为什么重要
8. 收口：本周最值得继续跟的一条线

## 03｜这件事为什么重要

### 标题
**Claude Code 这次更新，真正变的不是功能，而是接口**

### 核心判断
这篇要做的不是更新播报，而是告诉用户：

- 表面上像功能更新
- 实际上更重要的是工作接口变化
- 一个工具什么时候值得重视，不看功能点数量，要看它改了哪一层工作方式

### 为什么它适合做首页第 3 篇
因为它一上来就展示“不是报新闻，而是做转译”。

### 图卡页数建议
- **7 页**

### 图卡结构建议
1. 封面
2. 大家通常会怎么理解这次更新
3. 为什么那样看太表层
4. 真正变的是哪层接口
5. 这层接口为什么重要
6. 它会带来什么后续变化
7. 收口

## 04｜最近最值得看的变化

### 标题
**为什么最近大家突然都在讲 AI 知识库**

### 为什么它适合做首页第 4 篇
因为它既有“最近发生了什么”，也能补上“为什么现在是它”。

### 图卡页数建议
- **7 页**

### 图卡结构建议
1. 封面
2. 最近知识库内容为什么突然变多
3. 它和上一轮 AI 内容热词有什么不同
4. 为什么这个时间点会冒头
5. 它解决的是哪类问题
6. 普通人现在该怎么理解
7. 收口

## 05｜这件事为什么重要

### 标题
**MCP 开始进入争论期，这反而说明它走到了第二阶段**

### 核心判断
这篇是典型的“变化判断卡”：

- 早期讲 MCP 的内容大多在科普
- 现在开始出现路线分歧、价值判断、值不值得折腾
- 一个概念从普及期走到争论期，通常意味着它开始进入真实应用阶段

### 为什么它适合做首页第 5 篇
因为这篇很符合“信息账号”的节奏：抓最近变化，再给结构判断。

### 图卡页数建议
- **7 页**

### 图卡结构建议
1. 封面
2. 早期 MCP 内容的主叙事
3. 最近 MCP 内容的变化
4. 为什么会从科普走向争论
5. 争论说明了什么
6. 普通人现在该看什么
7. 收口

## 06｜变化背后说明了什么

### 标题
**最近 Agent 内容有一个明显变化：开始从概念转向分工**

这篇不是讲 Agent 是什么，而是讲内容生态的变化：

- 以前大家在问 Agent 是什么

### 为什么它适合做首页第 6 篇
因为它能把“内容变化”转成“阶段判断”，很像一个懂行的信息号。

### 图卡页数建议
- **7 页**

### 图卡结构建议
1. 封面
2. 早期 Agent 内容的典型问题
3. 最近 Agent 内容开始问什么
4. 为什么这是一个重要变化
5. 说明它走到了哪一阶段
6. 普通人现在怎么跟
7. 收口

## 07｜最近最值得看的变化

### 标题
**最近 Skills 这条线，哪些是真的进了工作流，哪些只是热闹**

### 图卡结构建议

4. 为什么热闹型不够
5. 哪种是真的进入工作流
6. 怎么判断一个 Skill 值不值得继续看
7. 普通人怎么选
8. 收口

## 08｜补一张背景卡

### 标题
**为什么大家现在都把 Memory、Knowledge Base、Skill 放在一起讲**

### 核心判断
这篇不做完整教材，只做最小背景补充。

重点是：

- 最近这三个词经常一起出现
- 不是因为它们是一个东西，而是因为它们分别卡在“记住、沉淀、复用”三层
- 现在一起被讨论，说明大家开始关注长期工作结构

### 为什么它适合做首页第 8 篇
因为这类背景卡是必要的，但只能作为辅助，不应该先于变化卡出现。

### 图卡页数建议
- **6 页**

### 图卡结构建议
1. 封面
2. 为什么最近总同时看到这三个词
3. Memory 负责什么
4. KB 负责什么
5. Skill 负责什么
6. 收口：为什么现在它们会被一起讲

## 09｜这件事为什么重要

### 标题
**这轮 AI 变化里，我为什么开始少看模型榜，多看工作接口**

### 核心判断
这篇是“个人判断型信息卡”，不是心情分享。

重点讲：

- 过去看模型榜很合理
- 但最近真正影响工作方式的，不是参数涨多少，而是谁在补接口、记忆、交接、工作流层
- 如果只看模型，不看接口，很容易错过更重要的变化

### 为什么它适合做首页第 9 篇
它能让账号多一层“人”，但这个“人”是在做判断，不是在输出日常。

### 图卡页数建议
- **6 页**

### 图卡结构建议
1. 封面
2. 为什么以前大家都盯模型榜
3. 为什么现在这个视角开始不够
4. 我现在更看什么
5. 这个切换为什么重要
6. 收口

## 10｜最近最值得看的变化

### 为什么它适合做首页第 10 篇

3. 变化 2 + 为什么值得懂
4. 变化 3 + 为什么值得懂

## 11｜变化背后说明了什么

### 标题
**AI 工具最近在补的，不是能力，而是工作结构**

### 核心判断
这篇是结构判断卡。

重点讲：

- 最近很多变化表面看起来分散：Memory、Skill、MCP、Agent、KB
- 但它们共同补的不是“再聪明一点”，而是“更能进入工作结构”
- 这是这轮变化里最值得被看懂的主线

### 为什么它适合做首页第 11 篇
它能把前面几篇零散变化收拢成一个大的结构判断。

### 图卡页数建议
- **7 页**

### 图卡结构建议
1. 封面
2. 看起来分散的几条变化
3. 它们表面上分别在补什么
4. 底层共同补的其实是什么
5. 为什么这条主线重要
6. 对内容跟踪意味着什么
7. 收口

6. 最近 Agent 内容有一个明显变化：开始从概念转向分工
7. 最近 Skills 这条线，哪些是真的进了工作流，哪些只是热闹
8. 为什么大家现在都把 Memory、Knowledge Base、Skill 放在一起讲

- **8-12**：补必要背景与更大结构判断

如果现在只先做 3 篇，我建议先发这三个：

#### 为什么先发
这是整个账号的总封面，既有判断，又不老套，还能把账号和普通 AI 资讯号区分开。

**Claude Code 这次更新，真正变的不是功能，而是接口**

#### 为什么先发

### 先发 3
**为什么最近大家突然都在讲 AI 知识库**

#### 为什么先发
这篇抓的是一个真实正在发生的内容信号，而且非常接近平台当前兴趣带。

这三个组合起来，正好覆盖：

- 母判断
- 单点变化转译
- 热线信号解释

## 跟 v1 的区别

v1 更像：

- 一套按章节展开的 AI 工作流知识号
- 偏 evergreen
- 偏“讲清楚一套东西”

v2 更像：

- 一个变化驱动的信息转译号
- 偏 recent signals
- 偏“抓住变化并解释为什么重要”

如果后面要继续做，这个账号更应该优先沿着 v2 走。

## 下一步

在 v2 基础上，后续最自然的不是继续列标题，而是：

1. 先把首发 3 篇做成图卡提纲
2. 给这 3 篇各出 2 版封面标题
3. 再定一个固定的“周变化精选卡”模板

这样这个号才会真正长成“持续转译变化”的内容系统，而不是回到旧的知识沉淀路线。
