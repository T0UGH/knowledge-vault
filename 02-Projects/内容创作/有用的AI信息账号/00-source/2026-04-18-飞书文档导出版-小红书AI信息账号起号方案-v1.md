# 小红书 AI 信息账号起号方案 v1

## 先说结论

这个号不建议继续做成“一个人在分享自己怎么用 AI 的个人博主号”。

更适合做成：

> **一个持续解释 AI 工作方式变化，并给出真实最佳实践的信息账号。**

它的任务不是陪伴，不是表演，不是追热点，而是：

- 帮人看懂变化
- 帮人理解值不值得跟
- 帮人搭出更稳的工作流
- 帮人把 Agent / Memory / Knowledge Base / Skill 这些概念接回真实工作

---

## 一、账号定位

### 一句话定位

> 帮你看懂 AI 工作流、Agent、知识库和工作方式背后的真实变化，并给出普通人现在就能用上的判断与做法。

### 用户看到这个号之后，脑子里应该留下的一句话

> 这个号能帮我省时间、少踩坑、看懂什么真的值得学。

### 账号角色

这个号更像：

- AI 工作方式观察员
- Agent / 外部大脑 / 工作流转译者
- 最佳实践编辑部

而不是：

- AI 情绪博主
- AI 日常陪伴号
- AI 热点快讯号

---

## 二、起号思路

### 1. 起号目标不是先追一条大爆

前期最重要的不是“爆”，而是让平台和用户都知道：

> **你这个号到底稳定输出什么。**

所以前 15 到 20 篇最怕的不是数据一般，而是：

- 题材太散
- 风格乱跳
- 一会像媒体，一会像教程，一会像个人感受帖

### 2. 正确打法：固定栏目 + 固定标题结构 + 固定视觉系统

前期要先把账号味道做出来。

建议用：

- 固定栏目
- 固定封面格式
- 固定标题句型
- 固定内容边界

这样用户点进首页后，会很快知道这是个什么号。

### 3. 前期内容比例

建议按这个比例走：

- **40% 判断型内容**：讲“为什么重要”
- **40% 工作流型内容**：讲“怎么长期用”
- **20% 轻个人验证内容**：讲“我现在怎么用、为什么这么判断”

### 4. 这个号要牺牲一点活人感，但不能完全没人味

建议保留的“人味”只有这几种：

- 我现在怎么判断
- 我踩过什么坑
- 我为什么改成这套结构
- 这套方法是我自己长期在跑的

但不要把主轴做成：

- 日常表达
- 个人情绪
- 陪伴感内容
- 大量“我”的状态输出

这个号更应该像：

> **一个可信的长期信息源。**

## 五、首页栏目设置

前期首页不要太花，建议稳定成 4 个视觉栏目。

### 栏目 1：这次变化，真正重要在哪

这是判断型栏目，适合发：

- 某次更新为什么重要
- 某个概念为什么很多人理解反了
- 某个机制为什么进入下一阶段
- 某种工作方式为什么开始成立

### 栏目 2：真实可用的工作流

这是最佳实践型栏目，适合发：

- 一个能长期用的 AI 工作流怎么搭
- 工具到底怎么分层使用
- Memory / Knowledge Base / Skill 怎么接起来
- 哪些方法是真的进入工作结构了

### 栏目 3：一篇讲明白

这是转译型栏目，适合发：

- MCP 到底是什么关系，不是什么神话
- Skill 和 prompt 到底差在哪
- 外部大脑和第二大脑到底差在哪
- Agent 为什么不是自动化脚本升级版

### 栏目 4：我现在怎么用

这是轻个人验证层，适合发：

- 我现在怎么判断工具分工
- 我最近为什么改用某套结构
- 我放弃了哪些看起来很热闹但不长期的东西
- 我为什么不再把 AI 当成一个单点工具

### 首页建议比例（前 12 篇）

- 4 篇「这次变化，真正重要在哪」
- 4 篇「真实可用的工作流」
- 2 篇「一篇讲明白」
- 2 篇「我现在怎么用」

## 六、封面与视觉建议

这个号不适合太萌、太生活、太日常。

更适合：

- 白底 / 浅灰底 / 轻纸面感
- 黑字为主，少量强调色
- 一张封面只放一个主判断
- 标题短，副标题补充收益
- 保持统一栏目头

### 封面建议结构

#### 顶部：栏目标签
例如：
- 这次变化
- 工作流
- 一篇讲明白
- 我现在怎么用

#### 中部：主判断
例如：
- MCP 真正改变的不是“连工具”，而是工作边界
- 一个能长期用的 AI 工作流，至少要有 4 层
- 很多人其实理解错了 Skill

#### 底部：收益补充
例如：
- 给你一个判断框架
- 一篇讲明白
- 我现在的真实做法

---

## 七、标题规则

建议统一采用：

> **对象 + 判断 + 收益**

### 推荐句型

- 不是 X，而是 Y
- 很多人其实理解反了
- 一篇讲明白
- 这次真正变的不是功能，而是接口
- 值不值得折腾？我给一个真实判断
- 从“会用”到“能接进工作流”，差的到底是什么

### 示例标题

1. **Claude Code 这次更新，真正变的不是功能，而是接口**
2. **很多人其实理解错了 MCP：它的价值不只是“连工具”**
3. **Memory、Knowledge Base、Skill，到底是什么关系？一篇讲明白**
4. **一个能长期用的 AI 工作流，至少要有 4 层**
5. **如果你只想跟一类 AI 变化，我建议你盯“工作方式”而不是“模型榜单”**

## 八、内容边界

### 应该多发的

- AI 工作流
- Agent 使用结构
- 外部大脑 / Memory / Knowledge Base
- Skill / MCP / 工作接口
- 新机制的价值判断
- 长期有效的最佳实践

### 可以发，但只适合少量补充的

- 个人感受
- 跟自己工作方式有关的经验
- 某次具体踩坑
- 世界观内容

### 不建议作为主轴的

- 泛新闻
- 泛模型榜单
- 泛行业快讯
- 大量日常人格输出
- 太细碎的单功能教程
