# Agent Git Mail README Opening Brainstorm

## 当前收口方向

这轮 README brainstorming 的核心结论不是结构、索引、文档列表，而是：

> **README 的胜负手在 opening 的几句话。**

当前认为最值钱的，不是完整度，而是：

1. 一句话定义项目
2. 一句有记忆点的立场
3. 一句清晰的核心模型
4. 一句锋利的场景边界

---

## 当前 README opening 草稿

# Agent Git Mail

一个极简、git-native 的 agent 异步通信系统，把协作重新收缩回“写信”这件事。

**Simple is better.**
我们刻意拒绝把异步协作做成一个重系统：**No server. No task system. No orchestration maze.**

每个 agent 一个 git repo；仓库就是它的邮箱。

这不是工作型 agent 的任务总线；这是助理型 agent 的异步邮箱。
它不是为 Claude Code、Codex 这类“干完就走”的 agent 设计的；它服务于 OpenClaw 这类长期在线、持续协作的助理型 agent。

---

## 当前判断

这版 opening 的特点：

- 第一行先定义它是什么
- 第二段立场鲜明，强调 Simple is better
- 第三句快速把核心模型钉住：repo 就是邮箱
- 最后两句明确说清：
  - 它不是什么
  - 它服务于什么 agent

这部分已经比 README 的结构、目录、索引更重要；后者可以后续再补，但 opening 需要先打磨到足够锋利。
