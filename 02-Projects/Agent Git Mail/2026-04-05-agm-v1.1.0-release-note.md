# AGM v1.1.0 release note

日期：2026-04-05
状态：released

- package: `@t0u9h/agent-git-mail`
- version: `1.1.0`

## 发布结论

AGM `v1.1.0` 已完成发布。

这次版本的意义，不是加了一批花哨功能，而是把 AGM 从“已经能跑的 MVP”进一步收成一个**边界更清楚的 1.0+ transport**。

如果用一句话概括这次版本：

> `v1.1.0` 让 AGM 在 contract、CLI、daemon、doctor 和 README 之间更一致了。

## 本次发布包含的核心变化

### 1. 1.0 contract closure 落地

这次版本最核心的是 AGM 1.0 contract closure 的实现收口，包括：

- `filename` 作为 canonical message id 的语义更明确
- `send` / `reply` 显式区分：
  - `local success`
  - `delivery success`
- partial failure 被正式提升为一等结果
- daemon first-run 改成 waterline-only，不 replay 历史 wakeup
- activation retry / checkpoint 语义进一步对齐
- doctor 对 activation / repo failure 的诊断能力增强

### 2. README 与运行时真相重新对齐

这次还同步更新了 README：

- 明确 `filename = canonical message id`
- 明确 delivery semantics
- 明确 daemon first-run 语义
- 当前状态更新为：1.0 contract closure 已完成

这是一次很重要的文档收口，因为 AGM 之前一个明显问题就是 README 容易把系统讲得比它实际边界更大。

### 3. 验证通过

本次发布前的验证结果：

- **typecheck passed**

- **npm pack dry-run passed**
- **npm publish succeeded**

## 仍然保留的后续项

这次发布不意味着 AGM 没有后续工作。

当前更合理的后续方向是 1.1.x / 1.2.x 级别的精细化，而不是重新发散系统边界。已明确的尾项包括：

- git error classification 精细化
- event taxonomy 的进一步收口
- 必要时再补少量 adverse-case tests

这些属于 **polish / hardening**，不是当前版本发布的阻塞项。

## 一句话总结

AGM `v1.1.0` 是一个“收口型版本”：

> 它不是把 AGM 做得更大，而是把 AGM 做得更像一个能被认真理解和认真使用的系统。
