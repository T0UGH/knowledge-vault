# Claude Code 卷二架构设计草案 v1

## 一句话定位

卷二建议定为：

> **卷二：一次 Agent Turn 怎么跑起来**

它承接卷一已经建立起来的系统地图与对象地图，继续回答一个更动态的问题：

> **用户输入怎么变成一次完整的 agent turn？**

卷二的任务不是按源码目录逛 QueryEngine，也不是组件百科，而是：

> **把 Claude Code 主循环的动态主线真正拆开。**

---

## 读者必须带走什么

卷二最重要的目标，不是让读者记住更多函数名，而是让读者脑中留下：

> **Claude Code 主循环的稳定总图。**

读完后，读者至少要能复述：

- 请求怎样进入运行时
- 当前 query / 当前工作面怎样形成
- 系统什么时候只是回答，什么时候要触发能力
- tool use / action 之后，结果怎样重新回到当前 turn
- 一轮 turn 为什么会继续，什么时候又会收口

源码链路要有，但只是支撑，不是最终记忆点。

---

## 卷二的写作原则

### 1. 按时间顺序展开，不按组件拆

卷二的章节顺序应顺着一次请求的时间线展开：

- 请求进入
- 运行时接住
- 当前判断形成
- 是否触发能力
- 执行结果回流
- 这一轮继续或收口

### 2. 先总览，再逐段拆

卷二开头先有一篇总览，把整条动态主线亮出来；后面再逐段细拆。

### 3. 输入预处理只保留轻中等权重

输入预处理必须讲，但不能吃掉卷二主体。它只需要让读者知道：请求不是裸着进入主循环的。

### 4. 卷尾重新压成一张稳定运行图

卷二最后必须把前面拆开的主线重新压回一张稳定运行图，并顺手导向卷三、卷四。

---

## 建议粒度

卷二建议比卷一更细一点，但不要碎成源码切片。

推荐范围：

> **7~9 篇**

当前最推荐：

> **8 篇**

原因：

- 少于 7 篇，动态链路容易压糊
- 多于 9 篇，卷二会重新碎掉
- 8 篇最适合把主循环拆清楚，同时不抢卷三、卷四的活

---

## 建议的卷二 8 篇结构

## 01｜一次请求怎么进入 Claude Code 的主循环
- 卷二总起篇
- 把整条主线亮出来

**最该留下的判断**
> Claude Code 的主循环不是“收输入、回输出”的单步过程，而是一条把请求接入运行时、形成当前判断、触发能力、接回结果并持续推进的工作链。

---

## 02｜用户输入在进入运行时之前经历了什么
- 输入前站篇
- 解释为什么系统接住的不是原始裸输入

**最该留下的判断**
> Claude Code 接住的不是一段原始输入，而是一份经过整理、归位、归并后的运行请求。

---

## 03｜请求是怎么进入 QueryEngine 的
- 主循环入口篇
- QueryEngine 定位篇

**最该留下的判断**
> QueryEngine 不是普通函数入口，而是 Claude Code 把请求真正接入主循环的关键运行入口。

---

## 04｜当前 query 是怎么被组织起来的
- 当前工作面篇
- query 形成篇

**最该留下的判断**
> 当前 query 不是一段单独文本，而是带着上下文、messages 和系统约束形成的当前工作面。

---

## 05｜系统怎么决定这一轮要不要调用能力
- 决策分流篇
- 从主循环走向执行层的分水岭

**最该留下的判断**
> 在 Claude Code 里，模型的关键产出不只是答案，而是“下一步该不该做事、该做什么”的当前决策。

---

## 06｜tool use / action 之后，结果怎么重新回到当前 turn
- 结果回流篇
- 动态闭环硬化篇

**最该留下的判断**
> Claude Code 的闭环关键不在“调用过能力”，而在“执行结果重新进入当前 turn，成为下一步判断的输入”。

---

## 07｜一轮 Agent Turn 什么时候继续，什么时候收口
- turn 边界篇
- 卷二局部收束篇

**最该留下的判断**
> Agent turn 的边界不由“说了一句话”决定，而由当前结果是否已经足够收口决定。

---

## 08｜把整条主循环重新压成一张稳定运行图
- 卷二收尾篇
- 运行总图回收篇
- 对卷三、卷四导流

**最该留下的判断**
> 卷二真正建立的，不是几段源码链，而是 Claude Code 主循环的稳定运行图：请求如何进入、当前判断如何形成、能力如何被触发、结果如何回流、这一轮又如何继续或收口。

---

## 卷二和后续各卷的边界

### 不吃掉卷三
- 卷二讲“主循环如何走到执行”
- 卷三讲“执行层如何真正落地”

### 不吃掉卷四
- 卷二讲“主循环依赖什么工作面”
- 卷四讲“这个工作面为什么能长期维持、不失控、可恢复”

卷二真正的职责是：

> **先让读者把 Claude Code 的运行主线真正看见。**

---

## 一句话总结

> 卷二不是再解释一次 Claude Code 是什么，而是把“用户输入如何变成一次完整的 agent turn”这条动态主线真正拆开，并在卷尾重新压成一张稳定运行图，交给读者留在脑子里。
