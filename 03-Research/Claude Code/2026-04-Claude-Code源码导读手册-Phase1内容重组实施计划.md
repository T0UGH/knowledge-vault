# Claude Code 源码导读手册 Phase 1 内容重组实施计划

## 这篇看什么

这份文档不是讲 GitHub Pages 技术实现，而是把当前阶段真正要做的事情拆成一份**可执行的内容重组计划**。

当前阶段只做一件事：

> **先把现有 Claude Code 共读内容，重组成一套可以公开阅读的导读手册骨架。**

也就是说，这份计划只覆盖：

- 目录结构
- 映射表
- 导读层页面
- 主链文章最小轻修
- 91+ 并入规则

不覆盖：

- 站点技术栈
- GitHub Pages 配置
- 样式与视觉

## 先给主结论

这轮 Phase 1 的目标，不是“做站”，而是把内容整理到一个程度：

- 新读者知道从哪开始
- 老读者还能按旧编号检索
- 90 篇不再像研究日志，而像一套手册
- 后续新文章不会再把结构拉裂

所以这份计划的核心，是完成这 7 类产物：

1. 旧编号 → 新结构映射表
2. guidebook 总入口
3. 阅读路径页
4. 术语页
5. 91+ 并入规则页
6. 6 卷导读页
7. 主链文章最小轻修

---

## 一、项目边界

### 这一阶段要完成的
- 把 `00-90` 全部纳入新手册结构
- 给每篇文章确定新卷归属、卷内顺序与处理动作
- 补 guidebook 索引层页面
- 给主链文章补上最小导航层
- 定死 91+ 的并入流程

### 这一阶段不做的
- 不选 Docusaurus / MkDocs / Astro
- 不做 GitHub Pages 上线
- 不做大规模正文重写
- 不做品牌化首页

一句话说：

> **先把内容骨架做对。**

## 三、实施任务

## Task 1：冻结输入集并建立正式映射表

目标：把 `00-90` 这 91 篇正文作为固定输入集，建立新卷结构映射。

### 产出
- `guidebook/00-旧编号到新结构映射表.md`

### 要求
- 明确排除 `README.md`、目录总表等非正文文件
- 映射表字段至少包含：
  - 原编号
  - 原标题
  - 新卷
  - 卷内顺序
  - 角色
  - 处理动作
  - 备注
- 附带两份清单：
  - 需桥接说明的文章清单
  - 需弱去重的文章清单

### 完成标准
- `00-90` 全部出现
- 没有未归属文章
- 每篇都有角色和处理动作

---

## Task 2：创建 guidebook 总入口

目标：给整个手册提供一个真正可对外的起点。

### 产出
- `guidebook/README.md`

### 页面必须包含
1. 这是什么项目
2. 适合谁读
3. 从哪里开始
4. 6 卷结构总览
5. 推荐入口
6. 相关索引链接
7. 后续新增文章怎么并入

### 完成标准
- 首页首屏就能让人知道这是什么
- 新读者能快速找到“从哪里开始”
- 6 卷名字全部可见

---

## Task 3：创建阅读路径页

目标：降低新读者进入门槛。

### 产出
- `guidebook/01-最短阅读路径与完整阅读路径.md`

### 必须包含
- 最短路径
- 标准路径
- 完整路径

### 规则
- 最短路径 ≤ 12 篇
- 标准路径 ≤ 25 篇
- 每一步都要写一句“为什么在这里读它”

---

## Task 4：创建术语页

目标：降低高频内部词的理解成本。

### 产出
- `guidebook/02-术语总表-第一版.md`

### 至少覆盖
- QueryEngine
- runAgent
- system prompt
- context collapse
- compact
- microCompact
- snip
- sessionRestore
- MCP
- hooks
- plugin
- policy limits
- teammate runtime
- mailbox

### 规则
- 每个词 1-4 句中文解释
- 每个词至少带一个“延伸阅读”链接

---

## Task 5：创建 91+ 并入规则页

目标：防止后续新文章再次把体系拉裂。

### 产出
- `guidebook/03-91篇以后怎么并入手册结构.md`

### 必须写清楚
- 新文章继续保留连续编号
- 但不能直接按编号上主导航
- 必须先归卷
- 必须更新映射表
- 必须更新对应卷页
- 如影响入门路径，则更新阅读路径页
- 新增卷的门槛是什么

---

## Task 6：创建 6 卷导读页

目标：把 90 篇从“单篇集合”变成“手册章节”。

### 产出
- 卷一到卷六导读页

### 每卷必须包含
1. 这一卷在讲什么
2. 为什么这一卷重要
3. 前置知识
4. 建议阅读顺序
5. 本卷文章目录
6. 读完这一卷你会得到什么

### 规则
- 每篇列入文章都要有一句引导说明
- 各卷之间要有交叉链接

---

## Task 7：对主链文章实施最小轻修

目标：让主链文章至少具备公开阅读需要的导航层。

### 处理对象
- 映射表中 `角色=主链` 的文章

### 每篇至少补这 5 个元素
- 所属卷
- 原编号
- 上一篇
- 下一篇
- 推荐前置篇目（如有）

### 对 `轻修 / 弱去重` 文章额外要求
- 补一段 80-180 字导语，或
- 做一轮弱去重，删掉明显重复开场

### 不能做的事
- 不重写整篇主论证
- 不改核心判断
- 不把研究味磨平

---

## Task 8：回填设计稿状态

目标：让设计稿与实际产物保持同步，但不把设计稿写成 changelog。

### 要更新的文件
- `2026-04-Claude-Code源码导读手册-GitHub项目与GitHub-Pages设计稿.md`

### 只做这些事
- 增加“当前进度”区块
- 链接到 Phase 1 产物
- 标记哪些规则已经落地

---

## Task 9：把老 README 指向新入口

目标：让旧系列索引仍可用，但公开入口切换到 guidebook。

### 要更新的文件
- `cc源码共读笔记/README.md`

### 要求
- 增加一句明确的公开阅读入口说明
- 不破坏它原本作为旧系列索引的作用

## Task 10：最终验收与 readiness 判断

目标：不是只检查文件存在，而是确认这轮内容重组真的完成了。

### 验收清单
- 6 卷结构固定
- `00-90` 全部映射完成
- guidebook 首页存在

### 额外抽查
至少抽查 10 篇主链文章，检查：

- 推荐前置篇目

只允许给出这两个结论之一：
- `READY_FOR_SITE_PLANNING`
- `NOT_READY_FOR_SITE_PLANNING`

## 四、阶段 gate

### Gate A
**不完成映射表，不讨论站点侧边栏。**

### Gate B
**不完成卷导读和阅读路径，不做首页包装。**

### Gate C
**不完成主链文章最小轻修，不宣布内容结构完成。**

### Gate D
**不定义 91+ 并入规则，不宣布体系稳定。**

一句话：

> **先把内容结构做对，再谈展示层。**

---

## 五、成功标准

如果这轮 Phase 1 做对了，结果应该满足：

1. 新读者第一次打开就知道从哪开始
2. 90 篇不再像研究日志，而像一套手册
3. 老读者还能通过旧编号定位正文
4. 主链文章知道自己在整套体系里的位置
5. 新文章未来不会再把目录结构拉裂
