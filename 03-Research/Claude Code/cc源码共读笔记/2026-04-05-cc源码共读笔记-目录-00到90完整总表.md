## 61-63 Session / Resume 恢复链
- **61** conversationRecovery 是 resume 把会话重新接回 runtime 的恢复入口
- **62** sessionRestore 是把恢复包真正接回当前 runtime 的落地层
- **63** session 线收尾：Claude Code 怎么把会话存下来、读回来、再接着跑

## 64-68 MCP 专题
- **64** Claude Code 是怎么把 MCP 外部能力接进 runtime 的
- **65** MCPTool 调用链：外部 tool 是怎么被包装、调用、重试和治理结果的
- **66** 为什么在 Claude Code 里 CLI + skill 往往比铺太多 MCP 更实用
- **67** MCP 认证状态机：为什么 needs-auth 不是补丁而是正式能力层
- **68** MCP 权限边界：为什么外部能力进来以后还要再过统一闸门

## 69-72 Hooks 专题
- **69** hooks 总入口：Claude Code 是怎么把 hook 接进 runtime 的
- **70** PreToolUse 与 PostToolUse hook 是怎么进入工具执行主链的
- **71** SessionStart 与 Stop hooks 是怎么进入会话生命周期的
- **72** hooks 主线收尾：为什么 Claude Code 的 hooks 本质上是 runtime 编排层

## 73-78 Plugin 专题
- **73** Claude Code 的 plugin 系统到底在提供什么能力面
- **74** pluginLoader 是怎么把插件装配成运行时能力包的
- **75** plugin 的各能力接入面是怎么挂上去的
- **76** validate / schema / policy 为什么说明 plugin 不是随便加载的目录而是正式能力包
- **77** plugin CLI / install / marketplace 是怎么把 plugin 变成产品级生态对象的
- **78** 为什么说 Claude Code 的 plugin 本质上是统一扩展平台而不是几个零散扩展点

## 79-84 Permissions 专题
- **79** Claude Code 的权限系统到底在管什么
- **80** tool execution 和 permission decision 是怎么接上的
- **81** BashTool 的权限判断为什么不是跑命令前问一下这么简单
- **82** 路径权限、allow-deny 规则和 settings 持久化是怎么组成长期授权体系的
- **83** policy limits 是怎么把本地权限系统再往上套一层组织策略闸门的
- **84** 为什么说 Claude Code 的权限系统本质上是在给 agent runtime 做分层行动边界

## 85-90 Team / Swarm 专题
- **85** Claude Code 的 team / teammate runtime 到底在系统里处在什么位置
- **86** TeamCreateTool / TeamDeleteTool：team 是怎么被创建、注册和清理的
- **87** InProcessTeammateTask：真正的 teammate runtime 是怎么在同进程里跑起来的
- **88** mailbox + idle / shutdown 协议：teammate 之间是怎么通信和收尾的
- **89** LocalAgentTask / RemoteAgentTask 和 teammate runtime 的边界是什么
- **90** 为什么说 Claude Code 的 team 系统本质上是一个带 leader、mailbox 和 task runtime 的 swarm
