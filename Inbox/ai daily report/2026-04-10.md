# AI 日报（2026-04-10）

## GitHub 长期跟进

### Claude Code
- **2.1.98 是一版信息量很高的功能+安全修复更新：一边补上后台监控、Vertex AI 接入、脚本沙箱和 worktree/trace 能力，一边集中修掉 Bash 权限绕过，以及一批会话和权限系统的边角问题。**
- 全部变更点：
  - 新增交互式 Google Vertex AI 登录向导：在登录页选择第三方平台后，可以一路完成 GCP 认证、project/region 配置、凭证校验和模型 pinning。
  - 新增 `CLAUDE_CODE_PERFORCE_MODE`：在 Perforce 场景下，Edit/Write/NotebookEdit 遇到只读文件不再静默覆盖，而是直接失败并提示 `p4 edit`。
  - 新增 **Monitor tool**：允许后台脚本持续流式上报事件，适合盯日志、PR 评论、异步任务状态，不用再靠高频轮询刷 token。
  - 新增脚本子进程沙箱控制：`CLAUDE_CODE_SUBPROCESS_ENV_SCRUB` 打开后，在 Linux 上会启用带 PID namespace 隔离的 subprocess sandbox；同时新增 `CLAUDE_CODE_SCRIPT_CAPS`，用于限制单 session 的脚本调用次数。
  - print mode 新增 `--exclude-dynamic-system-prompt-sections`，目标是提升跨用户 prompt caching 命中率。
  - 状态行 JSON 输入新增 `workspace.git_worktree`，当当前目录位于 linked git worktree 内时会显式标出。
  - OTEL tracing 打通到 Bash 子进程：在启用 tracing 时向 Bash 子进程注入 W3C `TRACEPARENT`，方便把子进程 span 接到 Claude Code 的 trace tree 下。
  - LSP `initialize` 请求现在会通过 `clientInfo` 标识 Claude Code 自身。
  - 安全修复是一大重点：修掉了反斜杠转义 flag 被误判为只读、从而绕过 Bash 权限的漏洞；也修掉了 compound Bash command 绕过强制 permission prompt 的问题。
  - 继续收紧“看似只读”的 Bash 判定：带 env-var 前缀的命令、重定向到 `/dev/tcp/...` 或 `/dev/udp/...` 的命令、`grep -f FILE` / `rg -f FILE` 读取工作目录外 pattern 文件等，都不再轻易 auto-allow。
  - 修复多项权限系统状态一致性问题：`permissions.additionalDirectories` 可在 session 中途立即生效；移除 managed-settings allow 规则后无需重启；`--dangerously-skip-permissions` 也不会再被静默降级。
  - `/agents` 界面增强为 tabbed layout：Running tab 看实时 subagents，Library tab 可直接 Run agent/View running instance。
  - `/reload-plugins` 现在可在不重启的情况下拾取 plugin 提供的 skills。
  - `/resume`、`/export`、fullscreen URL copy、voice mode、hook 错误呈现、token usage 记录、VS Code Git Bash 检测等一串体验问题也一并修复。
  - 更新了 `/claude-api` skill，明确覆盖 Managed Agents。

### OpenClaw
- **2026.4.9 这版主线很清楚：继续补强记忆/做梦链路，同时加固权限与网关安全，也顺手修了一批会影响生产部署的边界问题。**
- 全部变更点：
  - Memory/dreaming 新增 grounded REM backfill lane，可通过历史 `rem-harness --path` 回灌旧 daily notes，把旧笔记重放进 Dreams 和 durable memory，而不是再维护第二套 memory stack。
  - Control UI 的 dreaming 视图更完整：加入结构化 diary 视图、timeline 导航、backfill/reset 控制、traceable dreaming summaries、grounded Scene lane 与 promotion hints，并提供 safe clear-grounded 动作，方便 staged backfill。
  - QA/lab 新增 character-vibes evaluation report，支持选模型、并行跑，便于 live QA 更快比较候选行为。
  - provider-auth 支持 `providerAuthAliases`：provider variant 可以共享 env vars、auth profile、config-backed auth 和 API-key onboarding 选项，减少核心代码里对各 provider 变体的硬编码。
  - iOS 版本管理改成显式 CalVer，`apps/ios/version.json` 被钉住；TestFlight 的短版本会一直沿用，直到维护者主动推进下一 gateway 版本，并新增 `pnpm ios:version:pin -- --from-gateway` 工作流。
  - Browser/security 修复 SSRF 隔离绕过：主 frame 在 click、evaluate、hook-triggered click、batched actions 驱动跳转后，会重新执行 blocked-destination safety checks。
  - `.env` 安全收紧：来自不可信 workspace `.env` 的 runtime-control env vars、browser-control override、skip-server 等变量会被阻止；危险 URL 风格 override specifier 会在 lazy loading 前就被拒绝。
  - remote node `exec.started` / `exec.finished` / `exec.denied` 事件被标记为 untrusted system events，且 node 侧提供的 command/output/reason 会先 sanitize，避免后续 turn 被注入伪造 `System:` 内容。
  - 不可信 workspace plugin 不再能在非交互 onboarding 中与 bundled provider auth-choice id 冲突，降低了把 operator secret 误导向不可信 plugin handler 的风险。
  - 依赖审计修复：`basic-ftp` 强制到 `5.2.1` 以修补 CRLF command injection；同时升级 Hono 与 `@hono/node-server`。
  - Android pairing、Matrix sync、Slack `url_private_download`、reply-run SecretRefs / OAuth reauth、session-history reload、Gateway chat 内部控制 token 泄漏、external route 被 announce follow-up 抢占等多个网关和渠道层问题都做了修复。
  - OpenAI provider 现在对未显式设置的 reasoning effort 默认补 `high`；Ollama 原生 `api: "ollama"` 路径在 `/think` 非 off 时可显示 thinking output。
  - Codex CLI 集成更新：OpenClaw system prompt 现在会透传到 Codex 的 `model_instructions_file`，让 fresh Codex CLI session 接收到与 Claude CLI 一致的系统提示。
  - Agents timeout/failover、Windows update build headroom、Plugin SDK 子路径导出、OpenRouter 带 `/` 的 model id 选择器保真等工程细节也同步修补。

## GitHub 趋势项目

- **oh-my-codex（Yeachan-Heo/oh-my-codex）**：本周趋势第二，属于 Codex CLI 的“工作流增强层”，不是替代 Codex，而是在外面补了一整套标准化执行流。README 给出的主线很直白：先 `omx setup`，再用 `omx --madmax --high` 启动，然后在 session 内用 `$deep-interview` 做需求澄清、`$ralplan` 做方案批准、`$ralph` 做持续推进，或者用 `$team 3:executor` 并行执行。它把角色、skills、hooks、`.omx/` 状态、tmux/worktree 团队运行时都打包成一个稳定工作层，也就不难理解为什么它在 coding-agent 圈里涨星这么快。

- **LiteRT-LM（google-ai-edge/LiteRT-LM） + Gallery（google-ai-edge/gallery）**：这两个项目今天值得放在一起看。LiteRT-LM 是 Google 开源的边缘侧 LLM 推理框架，README 主推 **Gemma 4 support** 与 `litert-lm run` CLI，可在 Linux、macOS、WSL、Raspberry Pi 等环境直接跑；同时支持 GPU/NPU 加速、多模态输入和 function calling。Gallery 则是面向终端用户的手机端体验层，已经把 Gemma 4、Thinking Mode、Agent Skills、Ask Image、Audio Scribe、Prompt Lab、Mobile Actions、模型管理与 benchmark 做成 Android/iOS 应用。对“本地跑 agent / multimodal / tool use”这条线来说，这已经不只是 demo，而是从 runtime 到 app 的整套 edge AI 叙事开始成型。

- **DeepTutor（HKUDS/DeepTutor）**：虽然是教育场景项目，但它的 agent-native 设计很值得继续跟。README 把 v1.0.0 描述为一次 ground-up architecture rewrite：统一 Chat / Deep Solve / Quiz Generation / Deep Research / Math Animator 五种模式到同一线程上下文里；提供 TutorBot 持久自治 tutor、CLI/SDK 入口、persistent memory、Guided Learning、Co-Writer 和 knowledge hub。最近 release 节奏也很密：4 月 4 日刚上 v1.0.0-beta.1，4 月 10 日已到 beta.4，新增 embedding progress tracking、HTTP 429 retry、跨平台 start tour 依赖管理、MIME 校验修复等，说明它不是“概念型 agent 教学产品”，而是在快速进入可用工程态。

- **Onyx（onyx-dot-app/onyx）**：这是一个更偏企业/团队使用的开源 AI 应用层。它不只做 chat UI，而是把 Agentic RAG、Deep Research、Custom Agents、Web Search、Artifacts、Actions/MCP、Code Execution、Voice Mode、Image Generation 这些能力放在同一平台里；README 还明确区分了 Lite 与 Standard 两种部署形态，并强调 50+ connectors 与企业能力（SSO、SCIM、RBAC、analytics、query history、whitelabeling）。如果接下来要持续观察“自托管 AI 工作台 / 企业 agent 平台”方向，Onyx 值得列入常驻观察名单。
