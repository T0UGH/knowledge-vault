### openclaw/openclaw

OpenClaw 今天更新很多，其中几条会直接影响产品能力和运维方式。

- 新增一等公民入口 `openclaw infer ...`，把模型推理、媒体生成、网页、embedding 等 provider-backed 推理工作流统一到 CLI hub，说明它在把“代理框架”继续往“通用 AI 运行时”推进。
- Google / Gemma 4 路线继续补强：一方面加入 Gemma 4 支持，另一方面补上显式关闭 thinking 的兼容语义，说明多模型兼容层还在细化。
- Memory 方向明显加码：
  - dreaming 开始 ingest 脱敏 session transcript，加入按日语料、checkpoint 与 promotion/doctor 支持；
  - `memory-wiki` 增加 claim/evidence、digest artifact、claim-health lint、矛盾聚类、staleness dashboard、freshness-weighted search；
  - memory prompt/context engine 暴露更多拼装接口。
  这说明它已经不只是把 memory 当成“存 markdown”，而是在往持续知识层、可维护信念层走。
- 安全修复很多，而且大多是底层约束：
  - ClawHub 插件安装增加 SHA-256 校验，缺少或损坏元数据时 fail closed；
  - `/allowlist add/remove` 需要 owner 级授权；
  - Feishu docx 上传开始严格尊重 workspaceOnly；
  - Browser SSRF、Gateway auth secret rotation、QQ/Teams 等链路继续补洞。
- 节点、审批、会话链路也有可用性增强：
  - sessions 增加 compaction checkpoint 与 UI branch/restore；
  - `host=node` 的 exec approval wrapper 逻辑更一致；
  - subagent 的 `lightContext: true` 终于真正贯通。
- README 同步收敛了对模型选择的表达：
  - 不再直接暗示某个特定旗舰模型是默认最佳；
  - 示例配置从硬编码 `anthropic/claude-opus-4-6` 改成 `<provider>/<model-id>`，强调“选你信任且已在用的 provider/model”。

**判断**：今天最值得跟踪的是两条主线：一是 `infer` 入口把 OpenClaw 往统一 AI runtime 推进；二是 memory/wiki/dreaming 这一组更新，说明它正在把“长期记忆”做成真正的知识基础设施。

## GitHub 周趋势

这条 lane 今天还是 agent / coding workflow 在拉动热度，但真正值得继续跟踪的项目没必要铺满一整页，重点看下面几类。

### NousResearch/hermes-agent

这是今天最值得继续盯的上升项目之一。它的核心叙事不是“再做一个 agent shell”，而是把 agent 做成**会成长、会积累技能、跨会话保留用户模型**的系统。这个方向正好打在当前 agent 产品最大的短板上：单次任务可能很惊艳，但上下文、偏好和能力留不下来。

为什么值得看：

- 它把“持久记忆 + 技能生长 + 多 agent 委托”打包成一个完整故事，而不是单点 feature。
- 如果这个方向跑通，agent 产品的竞争点会从“单次任务完成度”转向“长期使用后的复利”。
- 社区热度快速抬升，说明大家对“越用越懂你”的 agent 叙事确实有需求。

### Yeachan-Heo/oh-my-claudecode

这也是一个值得留意的项目，因为它不是单纯给 Claude Code 加脚本，而是明确押注 **teams-first 的多 agent 编排**。换句话说，它在尝试把 Claude Code 从“单人终端里的强助手”往“团队协作里的 agent orchestration 层”推。

为什么值得看：

- 方向上更贴近真实团队协作，而不是个人玩具式自动化。
- 如果它的任务拆分、角色分工、协作接口设计做得顺，可能会成为 Claude Code 周边生态的重要工作流层。
- 它和今天 X 上大量“多 agent 团队 / 协作平台”的讨论能互相印证，不是孤立热词。

### antinomyhq/forgecode

ForgeCode 的看点是“多模型 pair programmer”。现阶段，开发团队很少愿意被单模型供应商锁死，所以能把 Claude、GPT、Grok、Gemini、DeepSeek 甚至更多模型统一到一套 coding 交互里，天然就有市场。

为什么值得看：

- 它解决的是现实问题：不同模型各有长板，团队需要统一入口，而不是反复切工具。
- 如果它能把上下文管理、任务路由、配额/成本控制做好，会比“支持 300+ 模型”这个口号本身更有价值。
- 这类项目往往会成为企业内部落地 agent coding 的过渡层。

### google-ai-edge/gallery

这个项目的信号不在“coding agent”，而在**本地/端侧 GenAI 落地**。它强调的是让人直接试用本地模型和 on-device ML/GenAI 用例。在“隐私、成本、离线、响应速度”越来越重要的阶段，这类项目会持续吸引关注。

为什么值得看：

- 它降低了本地模型体验门槛，让“本地可跑”从概念变成可以试用的样例。
- 对开发者来说，它可能会成为观察端侧 AI 交互形态的样板库。
- 如果后续跟移动端、浏览器端、轻量推理框架联动，传播面会很广。

### Multica：把 Claude Code / Codex 拉进多人多 Agent 协作

@jiayuan_jy 正式介绍了 Multica，定位是“多人 + 多 Agent 协作平台”，并给出一段 20 多分钟的视频，覆盖产品定义、10 分钟快速上手，以及他们自己如何用 Multica 开发 Multica。这个信号的价值在于，它不是抽象地聊 multi-agent，而是在把 Claude Code/Codex 这类 coding agent 真正放进协作场景。

看点：

- 过去很多 agent 产品强调“一个人带一堆 agent”，但团队协作、任务归属、上下文共享并没有解决。
- Multica 试图把“人和 agent 混编协作”做成产品层能力，这比单纯的 prompt orchestration 更接近真实开发组织。
- “上线几天已有数千用户体验”也说明这个需求确实在爆发。

### Claude Code 资源清单继续外溢，生态学习成本开始产品化

@WEB3_furture 整理了一条很长的 Claude Code 资源串，覆盖官方文档、Partner Network、认证、MCP 服务器、技能目录、可视化 skills 浏览器以及各种 super list。信息本身未必全新，但它说明一个趋势：Claude Code 已经进入“新用户需要生态导览层”的阶段。

看点：

- 当一个工具开始出现“从 0 到精通”的大型资源汇总时，通常意味着周边生态已经复杂到不能只靠官方文档覆盖。
- MCP、skills、agent framework、资源导航站被并列整理，说明用户实际面对的是一个生态系统，而不是单体产品。
- 这对后续工具机会也很重要：教程层、导航层、评测层、配置托管层都会有空间。

### GitHub 趋势被 Agent 项目占榜，社区注意力继续向“可成长代理”倾斜

@GitTrend0x 把当天 GitHub 星标暴增项目拆成一条线程，重点提到 `hermes-agent`、`GitNexus`、`shannon`、`obsidian-skills`、`DeepTutor` 等项目。虽然帖子本身带有明显传播口吻，但它抓到的主线是可信的：今天开发者社区的注意力，仍在从“单模型能力”转向“带记忆、带工作流、带垂直能力的 agent 系统”。

看点：

- `hermes-agent` 代表可成长 agent；
- `GitNexus` 代表 repo 级图谱 + code intelligence；
- `shannon` 代表安全代理；
- `obsidian-skills` 代表知识库原生 agent 化；
- `DeepTutor` 代表长期陪伴式 tutor。  
这些项目虽然方向不同，但共同点很明显：AI 不再只是回答问题，而是在特定工作流里“接管一部分长期职责”。

## X 关注作者

今天关注对象里信息量最大的内容，集中在两条线：一条是 Anthropic 把 Claude Mythos Preview 只面向安全研究组织开放，另一条是 coding agent 周边插件和工作流还在快速迭代。

### Anthropic / Claude Mythos Preview：高能力模型先走“定向防守”发布

Anthropic 官方发布了 Project Glasswing，称其基于最新前沿模型 Claude Mythos Preview，目标是帮助保护全球最关键的软件系统；同时明确强调，这个模型找漏洞的能力已经超过绝大多数人类安全研究者。

围绕这次发布，@bcherny 和 @simonw 的讨论补上了两层关键信息：

- @bcherny 直接点出 Mythos 很强，也确实应该让人感到害怕，因此只对 cyber defenders 做负责任的 preview 是正确路径。
- @simonw 则把它解读为“能力已经超过 Opus，但先只给合作安全研究机构”，并认为在近期安全界警报不断的背景下，这是合理决定。

这件事的意义不只是新模型发布，更是一个更强的行业信号：

- 领先实验室正在把“高危高能力模型”的默认发布策略，从公开可用改成**场景限缩、机构限缩、用途限缩**；
- 第一个真正被当作 clear-and-present danger 的应用域，已经很明确是 cybersecurity；
- 对 agent/coding 工具生态来说，这会直接影响后续哪些能力能公开接入，哪些只能在封闭合作里试点。

### Codex 的 iOS 自动化插件继续长功能，agent 正在深入端侧开发流程

@Dimillian 提到 Codex 的 build iOS apps plugin 已更新：

- 自动安装并配置 XcodeBuildMCP；
- 补齐测试与 UI agentic automation 所需能力；
- 新增 App Intents skill，帮助把应用功能暴露给 iOS 系统能力。

这条信息量虽然不如 Mythos 大，但很值得跟：它说明 coding agent 不只是停留在“写代码”，而是在往具体平台工作流里钻，尤其是测试、UI 自动化、系统集成这些原本很碎、但价值很高的环节。
