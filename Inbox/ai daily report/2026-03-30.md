## x-feed

- **范式转移：“Harness Engineering” 正在取代 “Prompt Engineering”**
  近期 X 上出现大量关于 Harness 的讨论。清华与哈工大联合发表的新论文提出了 NLAH（自然语言 Agent Harness）与 IHR（智能 Harness 运行时），尝试将 Agent 容易失控的控制逻辑沉淀为可读、可移植的规则文件。不少开发者认为，Agent 任务失败往往不是因为模型能力不足，而是缺乏合适的反馈回路与操作环境护栏（Harness）。
- **Claude Code 落地实践：从对话转向系统化配置**
  社区开始改变 Claude Code 的使用方式：放弃把所有工作流塞进单个 `CLAUDE.md`，转而利用 `.claude/skills/` 目录进行颗粒度管控。例如，将安全审查写成独立的 `security-review` Skill 并只赋予只读权限。大学生“小八”撰写的“2 天 46 个文件重构 Claude Code”文章分享了具体实践，其中拆解的 Prompt Caching 三层缓存设计和多 Agent 协同思路提供了很好的参考。
- **从 Superpowers 到 Compound Engineering**
  以 Superpowers 为代表的框架解决了结构化调用 AI 的问题，社区讨论认为下一步的方向是“Compound Engineering（复合工程）”。其核心是将 AI 踩过的坑沉淀为记忆，让自我进化（Self-improving）的 Agent 在下一次 Session 中直接复用经验。

## x-following

- **前端文本排版底层创新：Pretext**
  Cheng Lou 开源了一个纯 TypeScript 文本测量库 [Pretext](https://github.com/chenglou/pretext)。它能在不操作 DOM、不触发浏览器回流（reflow）的情况下精确计算文本排版尺寸，速度比传统方案快近 500 倍。这有助于 AI 时代 UI 框架实时渲染大量动态文字瀑布流，支持在 120fps 下进行几十万个复杂文本框的虚拟滚动。
- **新奇 Agent Skills 涌现**
  - @GithubProjects 推荐了 `agent-reach` Skill，能让 Agent 免 API 费用接入 16 个互联网平台（包括 X、Reddit、YouTube、B站、小红书等）的搜索与读取权限。
  - @Dimillian 开源了 `review swarm` Skill，该技能可以在后台并发拉起 4 个子 Agent，分别对代码的“行为回归、安全风险、性能问题、测试覆盖”进行专项 Code Review。
