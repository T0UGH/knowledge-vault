# AI 日报（2026-04-02）

## GitHub 长期跟进

### Claude Code
- **这版重点不在大功能扩张，而是在 headless / hooks / MCP / 长会话稳定性上继续补齐可用性。**
- 全部变更点：
  - `PreToolUse` hooks 新增 `"defer"` 决策，headless 会话现在可以在工具调用处暂停，再用 `-p --resume` 让 hook 重新评估。
  - 新增 `PermissionDenied` hook，自动模式里如果分类器拒绝命令，可以返回 `{retry: true}` 让模型重试。
  - `@` mention 的 typeahead 现在会显示 named subagents，子代理调度更顺手。
  - 新增 `MCP_CONNECTION_NONBLOCKING=true`，`-p` 模式可以跳过 MCP 连接等待；`--mcp-config` 单个 server 的连接等待也被限制到 5 秒，不再被最慢的 server 整体拖死。
  - 自动模式里被拒绝的命令现在会出通知，并在 `/permissions` 的 Recent 里可直接重试。
  - `CLAUDE_CODE_NO_FLICKER=1` 提供无闪烁 alt-screen 渲染，配合虚拟化 scrollback，明显在修终端观感问题。
  - `Edit(//path/**)` / `Read(//path/**)` 的 allow 规则改为检查 symlink 解析后的真实目标，补了一层权限绕过风险。
  - Windows 侧修了几类实际使用问题：push-to-talk 组合键、voice mode WebSocket、CRLF 双写、Markdown 硬换行被吃掉。
  - 长会话和复杂输出相关的稳定性补丁很多：`StructuredOutput` schema cache 失败、LRU key 导致内存泄漏、50MB 以上会话删消息崩溃、LSP crash 后僵死、`/stats` 漏算 subagent token、prompt cache miss、nested `CLAUDE.md` 重复注入、autocompact thrash loop 等都被修了。
  - `--resume` / `--continue` 相关恢复流程继续补洞，包括 deferred tool input 超过 64KB、旧版 transcript tool result 导致崩溃等。

### OpenClaw
- **OpenClaw 这版是典型的大盘整：一边继续加面向聊天和平台集成的新入口，一边把 gateway / tasks / approvals / 路由这些基础层继续磨平。**
- 全部变更点：
  - `/tasks` 正式进入聊天界面，当前 session 可以直接看后台任务板、最近任务细节，以及 agent-local fallback 计数。
  - `web_search` 新增内置 SearXNG provider，host 可配，说明搜索后端继续往可替换、可自托管方向走。
  - Bedrock Guardrails 接进 bundled provider，说明企业侧安全和治理能力在补。
  - macOS 新增 Voice Wake 选项，可直接触发 Talk Mode。
  - Feishu Drive 评论事件流单独打通，支持评论线程上下文解析、线程内回复，以及 `feishu_drive` comment actions，飞书协作链路更完整。
  - `openclaw cron --tools` 加入 per-job tool allowlist，cron 任务的工具权限开始细粒度化。
  - `agents.defaults.params`、`agents.defaults.compaction.model`、`agents.defaults.compaction.notifyUser` 等默认参数继续补全，尤其 compaction 的提示和模型选择更可控。
  - provider-specific session grammar 下放到 plugin-owned session-key surfaces，继续整理多渠道会话路由层。
  - `/context detail` 现在会把 tracked prompt estimate 和 cached context usage 对比，额外提示 provider/runtime overhead。
  - tasks 相关修复很多：维护 sweep 不再把 gateway event loop 卡死；`/status` 和 `session_status` 隐藏陈旧完成任务；维护 sweep 会重新校验当前 task record，避免把活跃任务错误标 lost/pruned。
  - approvals 路径继续统一，Slack / Discord / Telegram 都走共享审批能力链路，保持 delivery 和 `/approve` 行为一致。
  - 多平台修复继续推进：Matrix 多账号 / bootstrap runtime、Discord ID 校验、BlueBubbles 配置、Telegram errorPolicy / cooldown、Gateway reload hash、Gateway pairing、session reset hooks 等都在收口。

### Codex
- **Codex 0.118.0 更像一次“把 CLI 真正带到复杂环境里”的版本。**
- 全部变更点：
  - Windows sandbox 新增 OS 级 proxy-only networking，不再只靠环境变量做出站限制。
  - app-server 客户端加入 ChatGPT device-code 登录流，浏览器回调不稳定时也能完成登录。
  - `codex exec` 支持 prompt + stdin 双输入工作流，适合脚本管道里边喂内容边下指令。
  - 自定义 provider 可以动态拉取和刷新短期 bearer token，不再只能吃静态凭证。
  - 首次创建项目本地 `.codex` 文件也纳入保护，补掉首次写入时可能绕开审批的口子。
  - TUI 恢复了几类缺失流程：hook 通知回放、`/copy`、`/resume <name>`、`/agent` 线程刷新、skills picker 翻页。
  - MCP 启动鲁棒性增强：本地 server 启动窗口更长，握手失败会重新显示 warning。

## Product Hunt 新品观察

- `Computer Use in Claude Code` 今天票数最高（408 票），卖点非常直接：把 Claude 的 computer use 能力接进 CLI，让它在 macOS 上能开 app、点击、输入、看屏幕。对 GUI 测试、视觉 bug 排查、只能靠桌面软件完成的自动化流程都很贴近实际。
- `Ollama v0.19` 的点也很清楚：Apple Silicon 上把推理底座改到 MLX，主打本地模型推理提速，同时补了 NVFP4、cache reuse、snapshots 和 eviction。对本地 agent / coding workflow 用户，这类底层吞吐和交互响应改进往往比“又上了一个模型”更实用。
- `traceAI` 是今天更偏基础设施的一条：直接把 LLM tracing 做成 OTel-native，覆盖 prompts、completions、tokens、retrievals、agent decisions，且能直接进 Datadog、Grafana、Jaeger 等现有可观测栈。对于不想为了 GenAI 单独再上一套 vendor 和 dashboard 的团队，这个方向很对路。
- `ClawMetry for NVIDIA NemoClaw` 继续在 agent runtime 可观测性上加码，主打一条 host 命令覆盖所有 sandbox，可实时看 thought、tool call、token cost、memory monitoring 和 flow visualization。哪怕它现在明显带营销味，但“sandbox 内部观测”已经是 agent 基础设施的稳定需求。
