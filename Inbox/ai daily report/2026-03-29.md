# AI 日报（2026-03-29）

## github-watch
本 lane 追踪核心 AI coding 及相关工具仓库的动态。

### anthropics/claude-code
- **v2.1.87 发布**：解决了 Cowork Dispatch 模式下消息无法送达的故障。
- **v2.1.86 发布**：在 API 请求中增加了 `X-Claude-Code-Session-Id` 请求头，方便代理层在不解析 Body 的情况下，直接按会话维度聚合请求。
- **v2.1.85 发布**：修复了子代理（subagents）无法访问 SDK 内置 MCP 工具的问题（主因是未同步到共享应用状态）。此外，处理了 Windows 环境下 `.bashrc` 的回归 Bug，并解决了因 transcript 文件损坏（`parentUuid` 循环引用）导致的启动挂起问题。

### openclaw/openclaw
- **2026.3.28 正式版发布**：本次更新包含多项不兼容变更（Breaking Changes）与核心功能迭代。
    - **Breaking**: 彻底移除旧版 Qwen Portal OAuth 认证，全面转向 Model Studio API；停止对两个月以上旧配置的自动迁移支持。
    - **xAI 增强**：内置 xAI 供应商迁移至 Responses API，支持原生 `x_search` 工具；`onboard` 流程现已支持自动配置 xAI 搜索插件。
    - **插件与审批流**：`before_tool_call` 钩子新增 `requireApproval` 异步支持。插件现在可以挂起工具执行，通过终端、Telegram、Discord 等渠道请求人工审批，并支持 `/approve` 统一命令。
    - **ACP 会话绑定**：支持在 Discord、iMessage 等频道直接调用 `/acp spawn --bind here`，将当前对话环境直接转为 Codex 支持的工作空间，无需额外跳转线程。
    - **MiniMax 更新**：新增 `image-01` 图像生成模型；清理了旧版 M2/M2.1/M2.5/VL-01 模型，目前仅保留 M2.7。
    - **其他优化**：改进了 Telegram 长消息分段逻辑（改为基于 HTML 长度搜索，而非粗略的字符估计）；修复了 WhatsApp 因“自对话”产生的无限回显循环；CLI 新增 `openclaw config schema` 指令用于快速查看配置定义。

### sst/opencode
- **v1.3.5 发布**：修正了插件钩子处理异步操作时的逻辑问题；优化了 GPT Prompt 结构，使其更精简并排除了文件引用方面的干扰。
- **v1.3.4 & v1.3.3 发布**：常规维护与琐碎修复。

### openai/codex
- **0.118.0-alpha.3 发布**：Alpha 阶段高频迭代。新引入了 `argument-comment-lint` 工具，并同步更新了各平台的二进制资源。

## product-hunt-watch
本 lane 关注 Product Hunt 上与 AI 编程、Agent 及工作流相关的热门新品。

- **CrabTalk (197 votes)**: 仅 8MB 大小的开源 Agent 守护进程。它能将 Agent 的全量事件（文本增量、工具调用、思考链路）实时流式传输给客户端。支持 One-curl 极简安装，允许用户接入自有模型。
- **Expect (109 votes)**: 让 Agent 在真实浏览器环境中测试代码。支持一键扫描未暂存的改动或分支差异，自动生成测试计划并执行。
- **Glance (85 votes)**: 为 Claude Code 打造的开源浏览器 MCP 服务。基于 Playwright 构建，提供 30 多个实用工具（涵盖页面导航、截图、表单填充等），支持端到端（E2E）测试与视觉回归测试。
- **RepoLens (113 votes)**: 代码库变更洞察工具，旨在帮助开发者快速理清代码库中的关键变化。
- **Aera Browser (221 votes)**: 专门针对自动化场景构建的 AI 浏览器。

- 飞书官方 CLI 正式开源。阿义（@AYi_AInotes）评价道：飞书此举通过 CLI 拆掉了 AI 与工作数据间的“界面墙”。Claude Code 现在能一句话完成安装与授权，直接操纵多维表格、发送消息或生成网页。这标志着国内 SaaS 向 AI 原生设计迈出了重要一步。

### 技术观点与工具动态
- **字节开源 Agent 沙箱**：字节跳动发布 AIO Sandbox，解决 Agent 运行环境的隔离难题。@turingou 认为当前版本偏重，设计重心更偏向环境本身而非 Agent 专用。
- **Harness Engineering**：讨论热度不减。核心观点在于“不要卷 GUI”，应通过 CLI 和结构化接口（Harness）让 AI 触达底层逻辑。
- **MCP 生态**：@steipete 发布 MCPorter v0.8.0，支持将 MCP 工具直接转换为 CLI 指令。
