# AI 日报（2026-03-31）

## GitHub 长期跟进

### Claude Code 2.1.88：自动模式、Hook、长会话稳定性继续补短板
Anthropic 今天更新了 `anthropics/claude-code` 的 `CHANGELOG.md`。这版最值得注意的，不是某个单一大功能，而是把最近快速扩张后最容易卡住真实工作流的几个点一起补上了。

先看新能力：
- 新增 `CLAUDE_CODE_NO_FLICKER=1`，可切到无闪烁的 alt-screen 渲染和虚拟滚动回溯，显然是针对长时间终端使用时的可读性问题。
- 新增 `PermissionDenied` hook，会在 auto mode 分类器拒绝命令后触发；hook 可以返回 `{retry: true}`，相当于给“自动模式被拒后如何继续尝试”留了一个可编排接口。
- `@` mention 的 typeahead 现在会显示 named subagents，说明 Anthropic 还在把“多子代理协作”继续往常规交互面板里收。
- thinking summaries 在交互式会话里默认关闭，需要 `showThinkingSummaries: true` 才恢复，方向上更像是把默认体验往“少噪声、少 UI 抖动”收。
- `/env` 现在会影响 PowerShell tool，不再只对 Bash 生效。

更重要的是，这版的稳定性修复很集中，尤其是长会话、结构化输出、缓存、Windows 文本处理这些真实高频问题：
- 修掉 long session 里 tool schema bytes 变化导致的 prompt cache miss。
- 修掉 nested `CLAUDE.md` 在长会话中被重复注入几十次的问题。
- 修掉 `StructuredOutput` schema cache bug，官方直接写明此前会导致多 schema 工作流约 50% 失败率。
- 修掉大 JSON 输入被 LRU cache key 长期持有导致的 memory leak，以及对超大文件 `Edit` 的潜在 OOM crash。
- 修掉 `--resume` 遇到旧版或中断 transcript tool result 时崩溃的问题。
- 修掉 CRLF 双倍写入、Markdown 末尾两个空格被吞、CJK/emoji 历史记录在 4KB 边界丢失等文本层问题。
- `/stats` 现在会计入 subagent/fork usage，不再低估真实 token 消耗。

整体看，这版不是“再发一个新奇命令”，而是在给 Claude Code 的长时运行、auto mode、subagent 和跨平台终端体验补地基。对重度使用者来说，价值比表面的功能点更大。

### Codex 0.117.0：插件工作流正式前台化，多代理 v2 和 app-server 路线继续推进
`openai/codex` 发布 `rust-v0.117.0`。这一版最核心的变化有三条主线。

第一条是**插件正式变成一等工作流**：
- Codex 启动时可以同步 product-scoped plugins。
- 新增 `/plugins` 浏览、安装、移除流程，认证和安装状态的处理更清晰。
- 官方把这套能力直接写进 New Features，而不是继续藏在 rollout flag 背后，说明插件生态已经从实验态进入默认路径。

第二条是**多代理 v2 更像可操作的系统，而不是内部实现**：
- sub-agents 现在使用 `/root/agent_a` 这种 path-based address，而不是难读的 UUID。
- 增加 structured inter-agent messaging 和 agent listing。
- remote multi-agent session 修复了 stale turn-steering race，并把 agent names 替代 raw IDs 展示出来。

第三条是**app-server/TUI 路线继续做厚**：
- app-server client 现在能发 `!` shell commands、watch 文件系统变化、连带 bearer-token auth 的 remote websocket server。
- prompt history recall 在 app-server TUI 中可用，且跨 session 保留。
- `/title` 终端标题配置同时支持 classic TUI 和 app-server TUI。
- 这一版还把 app-server-backed TUI 设成默认，并翻开了 plugin/app rollout flags。

其他值得记的细节：
- image workflow 更顺：`view_image` 返回 image URL，生成图可在 TUI 中重开，image generation history 支持 resume。
- 文档补了 exec-server、filesystem watch RPC、realtime transcript notification、Python `thread.run(...)` quickstart。
- 同时移除了 legacy artifact tool、旧 `read_file`/`grep_files` handler，继续收敛旧工具面。

如果把它和今天 X 上流出的 `codex-plugin-cc` 一起看，Codex 正在从“独立 CLI”转向“可嵌入别家代理工作流的能力层”。

### OpenClaw changelog：Matrix/Telegram/TTS/安全边界继续高频修补
`openclaw/openclaw` 当日 changelog 更新量不算特别大，但含金量很高，主要是在把群聊线程、媒体下载、TTS 诊断和安全边界继续往前推。

新增项里，比较值得留意的是：
- Matrix DM 支持按 `threadReplies` 细分线程回复策略，并保证从触发消息开始，thread session isolation 跟随有效 room/DM policy。
- TTS 增加 structured provider diagnostics 和 fallback analytics，说明语音链路问题开始被当作可观测系统来处理，而不是黑盒失败。

修复项里，和日常可用性最相关的包括：
- Telegram media 下载现在允许 RFC 2544 benchmark-range CDN 分辨率，避免 voice message、PDF 等附件报 `Failed to download media`。
- Telegram voice-note `.ogg` 在 `whisper-cli` fallback 前会先转码。
- sandbox browser image 安装 `fonts-noto-cjk`，截图里的中日韩字符不再出现 tofu box。
- memory FTS 增加 trigram tokenization 和 short-CJK substring fallback，中文/日文/韩文检索终于更靠谱。
- `web_fetch` 增加 trusted env-proxy path，但严格 SSRF fetch 仍走 pinned direct path，兼顾代理部署和目标绑定安全。
- host exec 环境不再允许请求侧覆写 Python package index 相关变量，避免包拉取被重定向。
- Gateway shared-auth rate limiting 在 WebSocket handshake 期间仍然有效，device token rotation 后会主动断开旧连接。

这批更新的共同点是：OpenClaw 还在持续把“多渠道消息系统 + 本地执行代理”最容易出事故的边角问题往工程化方向推。

### Claude Code 的 computer use 上线，CLI 开始真正接管桌面验证环节
今天 X 推荐流里信息量最大的一条，来自 `@claudeai`：**Computer use is now in Claude Code**。官方描述很直接：Claude 可以从 CLI 打开 app、点击 UI、测试它刚写出来的东西；目前以 research preview 形式面向 Pro 和 Max 计划开放。

### Codex 正式进入 Claude Code：插件化的双模型工作流开始成形
推荐流里另一条高信息量内容来自 `@reach_vb`：从今天开始可以在 Claude Code 里使用 Codex，命令是：

```bash
/plugin marketplace add openai/codex-plugin-cc
```

随后可用三个命令：
- `/codex:review`：只读代码审查
- `/codex:adversarial-review`：偏挑战式、对抗式审查
- `/codex:rescue`：让 Codex 接手任务

这条信号和 GitHub release、关注流长文说明互相印证后，能看出几个关键点：
- OpenAI 不是做一个独立桥接脚本，而是以官方身份把 Codex 变成 Claude Code 工作流里的插件。
- 目标定位非常明确：第二意见、挑战性审查、线程救援。
- `review gate`、后台运行、状态/result 管理这些配套已经有了，不是 demo 级接入。

这对 coding agent 生态是实质变化：模型竞争开始从“谁更强”转向“谁能更容易嵌入别人的主工作流”。
