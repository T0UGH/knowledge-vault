# 小红书分析服务定价与接入判断

## 这条记录回答什么问题

这条记录主要回答两件事：

1. 现成的小红书分析服务，公开付费和定价大概是什么情况
2. 如果要接到我们自己的研究流程里，现实可行的接入方式是什么

结论先放前面：

> 有现成服务，但公开定价透明度普遍不高；从“低成本试错 + 接入容易”来看，智小红最适合先试，新红 / 千瓜 / 灰豚更像平台级数据服务，适合后续按更重的研究或品牌投放需求再补。

## 1. 智小红

这是这轮里唯一直接查到公开价格表、且价格足够清楚的一家。

### 公开套餐价格

页面上直接展示：

- 普通会员：免费
- 基础版：¥26.80 / 月
- 专业版：¥46.80 / 月
- 企业版：面议

### 页面明确给出的核心能力

比较适合我们做研究的能力包括：

- 对标博主拆解
- 笔记批量下载
- 爆文笔记数据下载
- 博主数据批量下载
- 评论批量下载
- 蒲公英数据下载
- 视频 / 图片批量下载
- 商业数据查看
- 违禁词检测

### 页面明确给出的关键额度

研究相关的额度里，比较关键的是：

- 笔记详情获取
  - 免费：20条 / 天
  - 基础：1000条 / 天
  - 专业：2000条 / 天
- 笔记数据下载
  - 免费：30条 / 天
  - 基础：1500条 / 天
  - 专业：3000条 / 天
- 评论下载
  - 免费：200条 / 天
  - 基础：2000条 / 天
  - 专业：50000条 / 天
- 博主数据下载
  - 免费：20条 / 天
  - 基础：500条 / 天
  - 专业：2000条 / 天
- 查看笔记商业数据
  - 专业：1000条 / 月
  - 企业：20000条 / 月

### 安装与形态

它不是 API 优先产品，而是明显偏浏览器插件 + 工作台：

- Chrome 扩展
- Edge 扩展
- Firefox 扩展
- 自有工作台

### 我的判断

如果目标是先验证：

- 现成工具到底值不值
- 批量下样本是否比当前 MCP 高效
- 是否适合做 deep research 前置采样

那么 **智小红专业版（¥46.80/月）是最值得先试的**。

原因很简单：

1. 价格清楚
2. 试错成本很低
3. 批量下载能力对研究非常实用
4. 不需要一开始就走企业销售或复杂采购流程

## 2. 新红

### 已确认情况

- 存在“付费购买”页面入口
- 但实际访问时会跳转到新榜登录页 / 微信扫码登录
- 未拿到公开清晰套餐表

### 已确认能力方向

从公开页面与搜索摘要能确认它偏：

- 红人榜单
- 爆款笔记
- SEO 流量分析
- 品牌营销投放
- 笔记搜索与高级筛选
- 搜索结果支持导出（从搜索结果摘要可见）

### 我的判断

新红更像：

> 登录后查看套餐、或先试用、再销售转化的平台。

适合做：

- 赛道热度扫描
- 关键词 / SEO 方向判断
- 爆款池建立
- 达人 / 竞品初筛

但不太像一个开放 API、让你直接程序接入的产品。

## 3. 千瓜

### 已确认情况

- 官网公开展示很多能力
- Footer 中能看到“产品价格”字样，但未直接看到公开价格表
- 搜索结果里能看到一些会员与数据包片段，但没有清晰月费对照表

### 已确认能力方向

官网明确强调：

- 市场洞察
- 营销管理
- 内容创作
- 达人 / 笔记 / 品牌 / 品类数据
- 关键词趋势
- 品牌营销智库
- 定制报告
- 数据定制

### 我的判断

千瓜明显是更成熟、更偏品牌和投放团队的工作台。

更适合：

- 品牌投放
- 达人筛选
- 竞品观察
- 大盘趋势
- 结案 / 复盘

但如果只是个人或小团队做内容研究，**一开始就上千瓜不一定性价比最高**。

- 小红书只是覆盖平台之一
- 没看到公开价格表
- 页面可见：
  - 登录 / 注册
  - 商务合作
  - 咨询电话：0571 8735 9466
  - 邮箱：bd@huitun.com

### 我的判断

灰豚更像：

- 商业化 / 电商 / 带货 / 多平台投放决策服务

如果目标主要是：

- 小红书内容研究
- 关键词样本
- 复刻拆解

那它不是第一优先。

## 定价透明度对比

按目前看到的情况，可以粗分为三档：

### 高透明

- 智小红
  - 价格公开
  - 套餐与额度公开
  - 能快速判断值不值

### 中透明

- 新红
  - 能看到付费购买入口
  - 能确认很多功能方向
  - 但价格要登录后才更清楚

### 低透明 / 销售导向更强

- 千瓜
- 灰豚

这两类更像：

- 平台能力展示充分
- 但真正的价格、购买方式、企业权限，需要进一步登录或销售沟通

## 我们现实可行的接入方式

这里要明确区分“使用”和“系统接入”。

### 路线 A：人工使用 + 导出

这是最现实、最稳的一条。

适用于：

- 智小红
- 新红
- 千瓜
- 灰豚

方式：

1. 人工登录服务
2. 在网页 / 插件中筛选样本
3. 导出结果
4. 将导出数据接入自己的分析流程

优点：

- 落地最快
- 风险最小
- 不需要赌对方是否开放 API

缺点：

- 需要人工参与
- 自动化程度有限

### 路线 B：浏览器插件 / 工作台接入

这一条最适合智小红。

因为智小红本质上就是：

> 浏览器侧增强工具 + 自有工作台

这意味着最现实的接法不是 API 对接，而是：

- 安装扩展
- 登录账号
- 使用批量下载功能
- 把导出的数据交给自己的 pipeline 分析

也就是把它当：

> 研究链路里的“数据采集前端”

而不是把它当后端接口服务。

### 路线 C：企业销售 / 定制服务接入

更适合：

- 新红
- 千瓜
- 灰豚

这些产品更像：

- SaaS 工作台
- 销售支持
- 数据定制 / 定制报告 / 企业权限

现实接法通常会是：

1. 先拿试用
2. 确认是否支持导出、数据包、企业接口、API
3. 如果没有 API，就走“人工导出 + 内部分析”
4. 如果有 API，再评估系统集成成本

## 对我们最合理的策略

### 先试智小红

这是目前最值得先走的一步。

原因：

- 价格低
- 套餐清楚
- 功能与 deep research 前置采样高度匹配
- 试错成本远低于先谈平台级服务

### 保留当前 xiaohongshu-mcp

现有 `xiaohongshu-mcp` 仍然很有价值：

- 适合实时补查
- 适合对单个样本做详情验证
- 适合研究过程中随时机动取数

### 最终形成组合

更合理的组合不是“只买一个平台”，而是：

- 智小红：负责高效采集与批量下载
- xiaohongshu-mcp：负责实时补查与详情验证
- 自建 pipeline：负责真正的标题分型、结构拆解、评论聚类和复刻模板沉淀

## 当前建议

一句话总结：

> 如果现在就想动手，先试智小红专业版；如果后续研究范围扩大到品牌、投放、达人矩阵与大盘趋势，再补新红或千瓜。

按优先级排序：

1. 智小红专业版：低成本验证是否值得
2. 保持现有 MCP 链路：补机动验证能力
3. 后续再看是否需要新红 / 千瓜来补更高层的数据视角

这比一开始就重投入平台级服务，更适合当前阶段。
