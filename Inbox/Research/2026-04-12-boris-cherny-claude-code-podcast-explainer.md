# 当 Boris Cherny 说“编程正在被解决”，他到底在说什么？

如果你最近在关注 Claude Code，这期 Boris Cherny 的播客值得听。

不是因为它讲了多少产品功能，而是因为 Boris 和主持人一直在追同一个问题：**如果“写代码”本身正在被解决，软件行业接下来会重构什么？**

播客里最抓人的一段，是 Boris 讲自己的工作方式。

他说自己已经很久没有亲手改代码了，100% 的代码都由 Claude Code 完成。从去年 11 月开始，他一行代码都没亲手写过，但每天仍然会提交很多 PR，同时还有多个 agent 并行工作。

如果只看这几句话，很容易把它理解成生产力表演。但放回整期对谈里看，Boris 真正想强调的不是“我有多强”，而是：**工程师最重要的工作单元，正在从“亲手写代码”，变成“组织 agent 持续产出结果”。**

所以，这期播客真正讨论的重点，其实不只是代码生成。

在 Boris 的描述里，Claude Code 不只是补全工具，也不只是更强的聊天框，而是开始朝“像同事一样工作”的方向走。重点不再是单次生成代码，而是 agent 开始接触用户反馈、bug 报告、遥测数据这些更接近真实软件生产现场的信号，并据此推动下一步该修什么、做什么、发布什么。

这里就引出了这期播客里最激进、也最容易被误读的一句话：**编程正在被解决。**

如果只听这句话，它很像 AI 时代的口号。但顺着这期对谈去理解，它真正的意思不是“软件工程不重要了”，也不是“系统设计、稳定性、协作、验收这些问题都自动消失了”。它真正指向的是：

**把业务意图翻译成可执行代码，这件事的门槛正在快速下降。**

而且下降的速度，可能比很多工程师愿意承认的还要快。

如果按 Boris 这套叙事继续往下推，真正变化的是软件行业默认的稀缺性。

过去大家默认稀缺的是“会写代码的人”。但在 Boris 这套叙事里，未来更稀缺的，可能不再是“把代码敲出来”，而是另外几件事：

- 知道系统到底该做什么
- 能把模糊问题收口成清晰边界
- 能判断 agent 的产出到底对不对
- 能把多个 agent 组织成稳定流程
- 能在速度大幅提升之后依然守住质量

这其实也是主持人在播客里不断追问的方向。

总有人问，今天还要不要学编程？以后产品经理会不会被替代？工程师这个 title 会不会消失？在这些问题上，播客里的表达明显是故意往极端走的。像“人人都会写代码”“人人都会是产品经理”“软件工程师这个 title 会消失，最后只剩 builder”，这些话都很有传播性。

但这期播客真正值得听的，不是把这些话当成字面预测，而是看清它背后的方向：**很多岗位原来靠“谁会操作某种专业工具”维持的边界，正在被 AI 稀释。**

工具门槛一旦下降，岗位边界就会松动。未来当然不会真的只剩下一种岗位，但很多岗位的核心价值，都会从“亲手产出某个专业工件”转向“定义问题、约束结果、组织流程”。

所以，如果顺着这期播客继续往下推，它给工程师最大的提醒，其实不是“赶紧学会用 Claude Code”。

更重要的提醒是：**你接下来最值钱的能力，未必是继续把自己训练成更快的代码工人。**

真正更值钱的，可能是这些能力：

- 方案判断
- 架构收口
- 验收能力
- agent 协作能力
- 流程设计能力
- 对结果负责的能力

从这个角度再看 Boris 说的那句“我现在比以前更享受编程”，意思就更清楚了。

表面上看，他已经不再“写代码”了；但换个角度看，他反而更接近软件创造里真正核心的那部分工作：判断做什么、不做什么，组织谁来做，决定结果算不算好。

也就是说，写代码并没有消失。它只是越来越像软件生产链条里一个可以被下放、并行、自动化的环节。而人留在上面的部分，开始越来越靠近判断、编排和验收。

所以这期播客最值得认真对待的地方，不是哪一句耸动金句，而是它在悄悄改写一个默认前提：**未来区分强弱工程师的标准，可能不再是你能写出多复杂的一段代码，而是你能不能定义正确的问题，调动合适的 agent，建立可靠的流程，并最终交付一个真的能用、敢上线、可维护的结果。**

如果说过去的软件工程师，最怕的是自己写得不够快；那么接下来更大的挑战，可能会变成另一种：当代码已经可以被快速生产出来之后，你还有没有能力判断什么值得做，什么不值得做；什么能上线，什么不能上线；什么只是看起来像完成了，什么才是真的完成了。

所以这期播客真正值得反复听的，不是“工程师会不会消失”这种表层问题。

它真正重写的，是另一件更底层的事：**工程师工作的核心单元，正在变化。**

从“亲手把代码写出来”，变成“定义问题、组织 agent、验证结果，并对整个交付负责”。

游戏当然还没结束。

但这场对谈真正提醒我们的，是规则已经不再围着“谁写代码更快”来设计了。
