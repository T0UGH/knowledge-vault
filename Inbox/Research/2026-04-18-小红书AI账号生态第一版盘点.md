# 小红书 AI 账号生态第一版盘点

## 这份盘点是什么

这是一次**第一轮样本扫描**，目的不是立刻给出完整榜单，而是先回答四个问题：

1. 小红书上 AI 相关账号里，哪些类型已经明显成立
2. 这些账号的数据大概在什么量级
3. 他们最近都在写什么
4. 如果后面要做自己的内容线，应该往哪里切

这次我已经把本机的小红书链路接通：

- `xiaohongshu-mcp`
- `mcporter`
- 登录态

因此这次不是纯网页搜索，而是直接从平台内拿到：

- 搜索结果页笔记样本
- 单篇互动数据（点赞 / 收藏 / 评论）
- 账号主页信息（粉丝 / 获赞与收藏 / 简介 / 最近笔记）

## 本轮方法

### 关键词

第一轮先用了 3 个词做样本扫描：

- `Claude Code`
- `AI Agent`
- `AI编程`

### 抓取层级

本轮抓了两层：

1. **搜索结果层**
   - 标题
   - 作者昵称
   - 点赞 / 收藏 / 评论
2. **账号主页层**
   - 粉丝数
   - 获赞与收藏
   - 简介
   - 最近在发什么

### 限制

这还是**第一轮小样本**，不是全量爬取。

所以这份结果更适合拿来做：

- 内容类型判断
- 账号风格归类
- 选题方向感知
- 切入位判断

不适合拿来做：

- 严格榜单
- 精确排名
- 长周期趋势结论

## 从样本里能看出的内容类型

目前至少已经明确存在 4 条内容线：

### A. 工具 / 工作流最佳实践

例如：

- Claude Code 最佳实践
- Claude Skills 一篇就够了
- Claude Code MCP最佳实践
- 如何管理你的Claude Code项目

#### 这类内容的特点

- 收藏率高
- 更偏“能复用”
- 更适合图卡 / 清单 / 结构化讲解

---

### B. 新机制 / 新架构解读

例如：

- Claude的两种Multi-Agent架构，一篇讲明白
- Anthropic亲自下场做Harness，Agent要变天
- 一个8KB的文件，打爆了如日中天的Skills

#### 这类内容的特点

- 容易起讨论
- 适合“一个判断 + 一个解释框架”
- 不是搬新闻，而是做“为什么重要”的解释

---

### C. 作品感 / 实验感 / vibe coding

例如：

- 教 claude 画它自己
- 做语音交互应用
- 做 AI 视觉情绪板

#### 这类内容的特点

- 更小红书
- 更适合“过程 + 成果图 + 个人视角”
- 不一定最技术，但更容易被平台吃到

---

### D. 冲突型认知转译

例如标题里常见的：

- 还能这么好看？
- 一篇讲明白
- Agent要变天
- 不是CLI，是操作系统

#### 这类内容的特点

不是中性描述，而是带着：

- 反差
- 判断
- 升级感
- 冲突感

这一点非常重要，因为它直接影响点击和滑动继续率。

## 对后续内容线最有用的初步判断

如果站在“内容创作与发布官”的角度，当前小红书 AI 内容至少分成三路：

### 路线 A：媒体热点路

优点：
- 起量快
- 选题宽
- 容易吃大众流量

问题：
- 不是自己的内容资产
- 很容易变成追热点号

---

### 路线 B：教程 / 实用流

优点：
- 稳
- 收藏高
- 容易形成系列

问题：
- 很容易卷成“工具说明书”
- 同质化风险大

---

### 路线 C：内部材料 → 平台表达

这条最值得继续看。

也就是：

- 用内部 signals / 判断 / 研究做原材料
- 转成：
  - 一篇讲明白
  - 最佳实践
  - 为什么重要
  - 我实际怎么用
  - 这背后的结构变化

这条线的优势是：

> 不需要做第一个发现新闻的人，
> 只需要做最会转译的人。

## 对自己的内容切入建议（第一版）

如果后面继续做小红书 AI 内容，我目前判断最适合的不是走“纯媒体”或“纯工具测评”，而是优先做下面三条：

### 1. 工作流 / 最佳实践线

适合做：

- Claude Code / Agent / Skills / MCP / Memory 的实际工作流
- 哪种方法真的能长期用
- 哪些做法只是热闹，哪些开始进入工作结构

### 2. 结构变化 / 为什么重要线

适合做：

- 这次更新为什么重要
- 它改变的不是功能，而是工作接口
- 一条新机制为什么值得普通人看

### 3. 内部材料转译线

适合做：

- 把 Rook 的 signals
- MT 的判断
- Leo 的研究

转成平台表达：

- 一篇讲明白
- 一套图卡
- 一条口播
- 一个系列主题

## 下一步该怎么继续查

这份还是第一轮小样本，更像“方向感盘点”。

下一步可以继续往下做成一份完整的运营研究：

### 1. 扩大账号池

从 3 个词扩到更多相关词：

- Claude Code
- Cursor
- OpenAI Codex
- MCP
- Skills
- vibe coding
- AI工作流
- agent memory
- AI知识库
- AI自动化

### 2. 做账号分层

按账号类型分：

- 媒体型
- 个人实操型
- 产品实验型
- 解读型
- 讲师/课程型

### 3. 做标题结构统计

看高表现内容里常见哪些句型：

- 一篇讲明白
- 最佳实践
- 还能这么好看
- 不是 X，而是 Y
- 为什么它会改变什么

### 4. 找空位

最终回答这几个更关键的问题：

- 哪些已经卷了
- 哪些角度还有空位
- 更适合做系列还是单篇
- 哪条线最适合长期稳定输出
