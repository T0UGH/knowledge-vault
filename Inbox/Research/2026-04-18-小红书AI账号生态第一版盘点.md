## 这份盘点是什么

这是一次**第一轮样本扫描**，目的不是立刻给出完整榜单，而是先回答四个问题：

1. 小红书上 AI 相关账号里，哪些类型已经明显成立
2. 这些账号的数据大概在什么量级
3. 他们最近都在写什么
4. 如果后面要做自己的内容线，应该往哪里切

这次我已经把本机的小红书链路接通：

- `xiaohongshu-mcp`
- `mcporter`
- 登录态

因此这次不是纯网页搜索，而是直接从平台内拿到：

- 搜索结果页笔记样本

## 从样本里能看出的内容类型

目前至少已经明确存在 4 条内容线：

### A. 工具 / 工作流最佳实践

例如：

- Claude Code 最佳实践
- Claude Skills 一篇就够了
- Claude Code MCP最佳实践
- 如何管理你的Claude Code项目

#### 这类内容的特点

- 收藏率高
- 更偏“能复用”
- 更适合图卡 / 清单 / 结构化讲解

---

### B. 新机制 / 新架构解读

例如：

- Claude的两种Multi-Agent架构，一篇讲明白
- Anthropic亲自下场做Harness，Agent要变天
- 一个8KB的文件，打爆了如日中天的Skills

#### 这类内容的特点

- 容易起讨论
- 适合“一个判断 + 一个解释框架”
- 不是搬新闻，而是做“为什么重要”的解释

---

### C. 作品感 / 实验感 / vibe coding

例如：

- 教 claude 画它自己
- 做语音交互应用
- 做 AI 视觉情绪板

#### 这类内容的特点

- 更小红书
- 更适合“过程 + 成果图 + 个人视角”
- 不一定最技术，但更容易被平台吃到

---

### D. 冲突型认知转译

例如标题里常见的：

- 还能这么好看？
- 一篇讲明白
- Agent要变天
- 不是CLI，是操作系统

#### 这类内容的特点

不是中性描述，而是带着：

- 反差
- 判断
- 升级感
- 冲突感

这一点非常重要，因为它直接影响点击和滑动继续率。

## 对后续内容线最有用的初步判断

如果站在“内容创作与发布官”的角度，当前小红书 AI 内容至少分成三路：

### 路线 A：媒体热点路

优点：
- 起量快
- 选题宽
- 容易吃大众流量

问题：
- 不是自己的内容资产
- 很容易变成追热点号

---

### 路线 B：教程 / 实用流

优点：
- 稳
- 收藏高
- 容易形成系列

问题：
- 很容易卷成“工具说明书”
- 同质化风险大

---

### 路线 C：内部材料 → 平台表达

这条最值得继续看。

也就是：

- 用内部 signals / 判断 / 研究做原材料
- 转成：
  - 一篇讲明白
  - 最佳实践
  - 为什么重要
  - 我实际怎么用
  - 这背后的结构变化

这条线的优势是：

> 不需要做第一个发现新闻的人，
> 只需要做最会转译的人。

适合做：

- 把 Rook 的 signals

## 第二轮新增的 3 个明显信号

## 1. `vibe coding` 已经不是小概念，而是独立内容带

第一轮里虽然能看到一点 vibe coding 气息，但第二轮一加这个词，热度直接拉出来了。

而且这条线不是单一写法，它至少已经分成了三种：

### A. 指南型
例如：
- 这份 Vibe Coding 指南，把 AI 编程讲透了

这类内容适合：
- 图卡
- 教程长文
- “从概念到上手”结构

### B. 晒作品 / 晒案例型
例如：
- 别藏了！快晒晒你们用 Vibe Coding 搓的作品

这类内容适合：
- 社区感
- 评论区互动
- 展示型内容

### C. 审美人格化表达
例如：
- 同事都问我从哪领养的

这类内容看上去不是讲技术，但它借的是 AI 产物和人格化表达的势。

### 我的判断

`vibe coding` 在小红书上已经不只是一个技术术语，而更像一个：

> **既能讲方法，又能晒结果，还能带审美表达的内容容器。**

这说明如果后面做内容，不一定非要从“编程教程”切，也可以从“作品感 + 方法感 + 过程感”切。

---

## 2. `AI知识库` / `记忆` / `SKILL` 已经形成了稳定叙事

第二轮里最扎眼的一条不是 MCP，而是知识库。

例如：
- Karpathy AI 知识库经验分享 附SKILL
- 手把手教会你搭建出 Karpathy 同款 AI 知识库
- Hermes Agent 如何做记忆
- 从源代码看 Claude Code 如何做 Memory

### 这条线的特点

它不是纯工具测评，而是：

- 你怎么组织长期知识
- 你怎么让 AI 记住东西
- 你怎么把 SKILL / Memory / Knowledge Base 连起来

### 我的判断

这是一条非常值得重视的线，因为它天然适合：

- 图卡
- 一篇讲明白
- 最佳实践
- 结构化对比

而且它比“单纯换模型/换工具”更有长期复用价值。

如果后面做稳定内容线，`知识库 / memory / skill / 外部大脑` 是很值得重点占位的方向。

---

## 3. `MCP` 已经开始进入“观点争论期”，不只是科普期

第二轮里的 `MCP` 不是只有“什么是 MCP”这种入门内容。

它已经开始出现：

- MCP 陨落
- 手把手教会你什么是 MCP 协议
- 为什么能养龙虾 openclaw
- skill vs 需求概要详细设计文档

### 我的判断

这意味着 `MCP` 在平台里已经进入第二阶段：

1. 第一阶段：名词普及
2. 第二阶段：路线分歧 / 价值判断 / 应用分层

也就是说，光做“什么是 MCP”已经不够了。更有机会的，是：

- 为什么很多人理解错了 MCP
- MCP 到底值不值得折腾
- MCP、Skills、Agent 工作流之间是什么关系

## 现在可以更明确地说：AI 小红书内容至少有 6 条线

结合两轮样本，目前可以把内容线收成 6 条：

### 1. 热点资讯转译
- 媒体号更强
- 吃速度与标题

### 2. 工具 / 最佳实践
- Claude Code
- Skills
- MCP
- 工作流配置

### 3. 架构 / 机制解读
- Multi-Agent
- Harness
- MCP 路线争论
- 8KB 文件打爆 Skills 之类的认知转折题

### 4. vibe coding / 作品展示
- 展示成果
- 展示过程
- 展示“AI 让我能做什么”

### 5. 知识库 / Memory / Skill / 外部大脑
- 知识库搭建
- AI 记忆系统
- SKILL 体系
- 长期知识组织

### 6. 工作结构变化 / 为什么重要
- 这不是一个功能更新，而是工作接口变化
- 这不是一个工具，而是一套工作结构开始成形

## 对后续内容策略的更新判断（第二轮后）

### 最值得继续关注的 4 个主题簇

1. **Claude Code / Agent / Skills / MCP / Memory**
   - 仍然是主轴

2. **vibe coding**
   - 明显是平台友好型内容，不只是技术词

3. **AI知识库 / 外部大脑**
   - 收藏意图非常强，适合做高价值内容资产

4. **路线判断 / 观点型内容**
   - 比如 MCP 值不值得、某套架构到底有没有意义

## 下一步建议（第二轮后）

接下来不建议再只做“泛 AI 账号观察”，而应该进入更有用的阶段：

### 1. 做账号分层表
按类型列出：
- 媒体型
- 个人实操型
- 作品实验型
- 结构解读型
- 知识库 / 外部大脑型

### 2. 做标题句型库
例如高频句型：
- 一篇讲明白
- 最佳实践
- 还能这么好看？
- 不是 X，是 Y
- 为什么它会改变什么
- 手把手教会你

### 3. 做“适合自己切入的 3 条内容线”
基于两轮样本，下一步最适合直接收成：

- `AI 工作流 / 最佳实践`
- `知识库 / Memory / Skill / 外部大脑`
- `结构变化 / 为什么重要`

而 `vibe coding` 可以作为更偏平台感的侧翼，不一定要当主线，但值得借鉴其表达形式。
