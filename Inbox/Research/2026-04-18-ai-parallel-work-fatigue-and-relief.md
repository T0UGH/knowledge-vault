# 结论

- **这是正常现象，而且很常见。**
- 疲劳来源不是 AI 本身，而是：**并行调度、多上下文切换、结果验收、持续盯盘**。
- 本质上，负担从“自己做”转成了“持续管理多个半自动执行体”。
- 缓解方式不是靠意志力硬扛，而是要把协作方式从“即时对话驱动”升级为“批次化调度 + 结构化收货 + 明确收工线”。

# 为什么会更累

## 1. AI 放大了上下文切换成本

以前主要是：
- 一个 IDE
- 一个任务
- 一条实现主线

现在更像：
- 多个 agent 同时跑
- 多个终端 / chat / repo 状态并行
- 需要不停恢复：谁在做什么、做到哪、哪个结果可信、哪个要追问

真正累的不是“发一条消息”，而是：
- 恢复上下文
- 维持状态一致性
- 防止任务漂移
- 判断哪些结果值得继续跟

## 2. AI 带来了“低成本发起，高成本收尾”

AI 让开任务非常便宜：
- 起一个 agent 很快
- 再起一个也很快
- 同时推进多个方向也很快

但收尾成本没降，甚至更高：
- review
- 验收
- 对齐边界
- 纠偏
- 合并结论
- 决定哪些结果能信

所以会出现一种典型疲劳：

> 启动很爽，收尾很累；一直在推进，但一直收不了工。

## 3. 疲劳类型已经变了

这不再是传统 coding fatigue，更多是：
- **调度疲劳**
- **中断疲劳**
- **验收疲劳**
- **轻度持续认知过载**

换句话说，AI 没有消灭工作，而是把人的位置从执行位推向：
- dispatcher
- reviewer
- boundary owner
- final accountable person

这个角色更像项目经理 / 总工，天然更消耗工作记忆。

# 核心判断

> AI 放大的是系统产能，但人的调度系统如果没升级，疲劳会先升级。

所以问题不是“自己不够强”，而是：
- agent 数变多了
- 线程数变多了
- 但约束、节奏、验收、停机机制还停留在单线程时代

# 怎么缓解

## 1. 限制同时活跃 agent 数

建议：
- **同时活跃 agent 不超过 2 个**

不是不能开更多，而是：
- 2 个活跃
- 其他等待批次进入

经验判断：
- 第 3 个开始，收益通常下降
- 第 4 个开始，疲劳明显上升

如果没有强约束，并行通常不是加速器，而是噪音放大器。

## 2. 从“随时开任务”改成“按批次发任务”

不要零碎起任务。改成：
- 一次定义一个 batch
- 明确目标、边界、验收、停止条件
- 让 agent 跑完整段
- 自己只在批次边界介入

原则：

> 不持续盯盘，只按批次收货。

这会显著降低被 agent 持续打断的疲劳。

## 3. 强制状态交接模板

每个 agent 回报时，固定输出：
- 目标
- 当前状态
- 已完成
- 阻塞 / 风险
- 下一步
- 需要你拍板的点

这样做的价值不是“好看”，而是：
- 降低上下文恢复成本
- 降低重新通读对话的负担
- 降低 handoff 时的信息损失

## 4. 把自己从执行位拉回裁判位

最值钱的角色不再是“继续多做一点”，而是：
- 卡边界
- 卡验收
- 停掉低收益任务
- 做接受 / 延后 / 放弃判断

一句话：

> 不要同时做 5 个 agent 的同事，要做它们的 dispatcher + reviewer。

## 5. 设置 AI 收工线

必须有明确的“停止规则”，否则系统会一直拖着人转。

推荐规则：
- 晚上只允许收尾，不新开 agent
- 单个任务超过 2 轮纠偏还不收敛，就挂起
- 单个任务 30 分钟没有形成明确进展，就暂停
- 没有验收标准的任务不发出去

本质上是在给无限生成能力加一个人工截止器。

# 最小可执行规则

如果只想先做最小调整，可以直接上这 5 条：

1. **同时活跃 agent ≤ 2**
2. **每个任务都必须先写验收标准**
3. **每次回报都必须走固定 handoff 模板**
4. **晚上不新开 agent**
5. **超过 2 轮纠偏不收敛就挂起**

这 5 条的目标不是“更严格”，而是：
- 降低盯盘频率
- 降低切换次数
- 降低无效并行
- 降低收尾时的混乱感

# 推荐判断

当前更重要的，不是继续追求“同时调更多 AI”，而是升级自己的：
- 调度规则
- 状态交接格式
- 验收纪律
- 停机机制

如果这些不先升级，AI 越多，人越累。

# 下一步建议

如果继续往前推进，这个主题值得进一步沉淀为：

1. 一版个人 AI 并行协作减疲劳工作法
2. 一版固定 handoff 模板
3. 一版 agent 启动前检查单（目标 / 边界 / 验收 / 停止条件）

这样可以把“靠感觉抗疲劳”变成“靠流程减疲劳”。
