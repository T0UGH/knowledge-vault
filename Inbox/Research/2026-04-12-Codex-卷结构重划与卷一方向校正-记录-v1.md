# Codex 卷结构重划与卷一方向校正记录 v1

## 这轮最重要的判断

### 1. 当前 Codex 的中段卷结构切法不顺
核心问题不是“恢复机制写太多”，而是：

> **runtime core 没有独立成卷。**

当前切法里，卷二过早进入：
- rollout
- SQLite
- replay / recovery
- turn-history semantic projection

这会导致读者在还没真正看见“一条工作回合怎么跑起来”之前，就先被拖进持续性与恢复机制。

### 2. 新的中段卷结构建议
新的切法应改成：

- **卷二：runtime core / 一条工作回合怎么跑起来**
- **卷三：状态、持续性与恢复怎么成立**
- **卷四：控制面怎么把 runtime 暴露出来**

一句话压缩：

> **先讲它怎么跑，再讲它怎么续命，再讲它怎么被暴露出来。**

### 3. 卷一需要先给读者一个 preview，不然卷二卷三读不进去
这轮已经明确：

> **读者当前最大的问题，不是某一篇细节没解释清，而是脑中还没有 Codex 的整体图。**

所以卷一不能只是目录导览，而必须先完成：

1. 系统整体是什么
2. 主体在哪
3. 核心对象有哪些
4. 一条最基础的 interactive 主数据流 / 请求流怎么跑
5. 卷二、卷三、卷四分别挂在这张图的哪里

## 这轮对 Claude Code 的借鉴点

本轮重新确认：Claude Code 值得借鉴的不是“卷数相同”，而是它的认知坡度：

1. 卷一先建立系统全景与最小主链
2. 卷二正式讲一条 Agent Turn 怎么跑起来
3. 后续卷再展开上下文、执行、扩展、协作等层

对应到 Codex，本轮得到的稳定判断是：

> **Codex 也应该先建立系统 preview 和 runtime 主工作回合，再去展开恢复与控制面。**
