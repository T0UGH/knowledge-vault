### Body

突发奇想，测试一下现在的 AI 在数据可视化领域的“审美”到底如何。
  
于是我用 Claude Code 搓了一张 3D 人口密度图。结果出乎意料，没有任何复杂的调试，一次成型，效果蛮好
  
其背后的逻辑其实很简单：核心是 R 语言 + rayshader 包
  
我写了一个模块化的prompt，只需要把你想做的 国家/地区名字 填进去，代码就会自动拉取数据、裁剪边界、渲染出图。
  
这次最大的感受是：
1. 之前试过大部分AIGC产品，可以做出很美，但是幻觉很高的可视化。代码不仅赋予了数据“美感”，更保留了数据的“真实”。
2. AI 并不缺乏设计能力，真正的瓶颈在于高品味的引导。 拉开差距的不是提示词技巧，而是你的审美判断力，即能否清晰地定义“美”，并将其转化为 AI 能理解的语言。
  
关于指导AI生成美学可参考《分享4个方法，提升VibeCoding UI 质量》的帖子

### Body

TLDR版本：
1️⃣ Planning Tool / 计划与进度管理
核心思路：任务先拆解再执行，维护一份可变计划，按步推进，遇变则调。
解决什么：浅层 Agent 拿到任务就硬做，缺少检查点，一旦出错就偏离目标。
价值：把"项目管理"内置到 Agent 回路里，降低长任务失控概率，让每一步可解释、可调试、可回滚
  
2️⃣ Subagent Spawning / 隔离式深挖
核心思路：子任务交给独立Subagent深挖，只返回结构化摘要，主智能体上下文保持干净。
解决什么：把大量细节塞进主上下文会"污染"窗口，稀释核心目标，拖慢推理。
价值：分治 + 上下文预算管理。适用于大量资料调研、大代码库定位、多路线并行探索
  
3️⃣ Sandboxed Filesystem / 沙箱文件系统
核心思路：给 Agent 一个隔离工作区，用文件承载中间产物、关键上下文和可复用摘要。
解决什么：对话上下文会溢出，检索噪声大，纯内存式记忆无法支撑长链路任务。
价值：把短期记忆外化为长期可检索的工作区，Agent 从"chatbot"升级为"持续迭代的执行者"
  
4️⃣ Detailed System Prompts / 把提示词当操作手册
核心思路：Prompt 不是"你是个助手"这么简单，而是工程 SOP：何时计划、何时执行、失败如何恢复、输出格式如何约束。
解决什么：行为不可预测、不可调试、不可定制。
价值：一致性、可调试性、可定制性。不同应用场景对应不同工作规范。
