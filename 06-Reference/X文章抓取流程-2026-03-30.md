# X 文章抓取流程

> 创建时间：2026-03-30

---

## 核心方法

**用 agent-browser 的 Accessibility Tree 而非 OCR 截图**

`agent-browser snapshot -c` 返回的是浏览器内核生成的**无障碍树（Accessibility Tree）**，本质是渲染后 DOM 文本的层级结构整理，比 OCR 更快更准。

```
真实浏览器渲染页面
       ↓
浏览器内核生成 Accessibility Tree
       ↓
agent-browser 提取文本内容
       ↓
返回带层级结构的纯文本
```

---

## 完整步骤

### 1. 尝试 article 模式 URL
```
agent-browser open https://x.com/{user}/article/{id}
```
article 页面不需要登录，比 timeline 页面限制少。

### 2. 截取页面内容
```
agent-browser snapshot -c
```
`-c` 是 compact 模式，只返回文本内容和链接，去掉样式噪音。

### 3. 如果 article 也被拦截，尝试 timeline 模式
```
https://x.com/{user}/status/{id}
```
timeline 模式通常更严格，可能需要登录。

### 4. 保存为 Markdown
用 Write 工具写文件，保留原文结构（标题、列表、引用、引用块）。

### 5. 移动到 Obsidian 目录并 push
```bash
mv "文件" "vault/03-Research/AI/"
git add "文件"
git commit -m "添加: 标题"
git pull --rebase   # 远程有冲突时需要
git push
```

---

## 为什么不用 WebFetch

X 有 JS 渲染限制，WebFetch 拿到的是空壳（只有 CSS 和 JS 骨架）。

agent-browser 是真实浏览器，能执行 JS 渲染出真实内容，再通过 Accessibility Tree 提取文本。

---

## 适用场景

- ✅ **纯文字长文**（article 页面）— 直接读文本，最高效
- ✅ **有图片的推文** — 图片部分仍需 OCR 或手动处理
- ❌ **需要登录才能看的内容** — agent-browser 可以手动登录，但流程更复杂

---

## 关键工具

| 工具 | 用途 |
|------|------|
| `agent-browser open` | 打开 URL |
| `agent-browser snapshot -c` | 提取页面文本（compact 模式）|
| `agent-browser snapshot -i` | 提取可交互元素（需要点击/填写时用）|
| `agent-browser close` | 关闭浏览器 |
